**Part B — Breakpoint consolidation (per D-12):**

Consolidate the 5 existing media query blocks into 2 breakpoints:

1. Merge `@media (max-width: 900px)` (line ~974), `@media (max-width: 640px)` (line ~989), second `@media (max-width: 640px)` (line ~1005), and `@media (max-width: 700px)` (line ~1087) into a single `@media (max-width: 768px)` block containing ALL those rules.

2. Change `@media (max-width: 600px)` (line ~1630, botanical shrink) to `@media (max-width: 480px)`.

The merged 768px block should contain (in order):
- `.app-header { flex-direction: column; align-items: flex-start; }`
- `html[dir="rtl"] .app-header { align-items: flex-end; }`
- `.issue-grid { grid-template-columns: 1fr; }`
- `.app-shell { padding: 1.5rem 1rem 3rem; }`
- `.app-nav { width: 100%; }`
- `.session-edit { opacity: 1; transform: translateY(0); }`
- All the `.table` stacked card rules (thead clip, tr.client-row block display, td data-label, etc.)
- `.client-spotlight { flex-direction: column; align-items: flex-start; }`
- `html[dir="rtl"] .client-spotlight { align-items: flex-end; }`

**Part C — Global 44px touch targets (per D-11):**

Add inside the `@media (max-width: 768px)` block:

```css
/* Global 44px tap targets (Apple HIG) */
button,
[role="button"],
a:not(.app-footer-links a):not(.app-footer-contact a),
select,
.nav-link,
.header-control-btn,
.row-actions button,
.modal-close {
  min-height: 44px;
  min-width: 44px;
}
```

Note: Footer links excluded from 44px min-width to avoid oversized footer. Severity buttons get their own sizing in Plan 02 (D-08 says 36-40px for severity specifically).

**Part D — Body scroll lock class (per D-04):**

Add AFTER the `:root` z-index tokens, near the top of app.css:

```css
/* Body scroll lock — prevents scroll-behind on mobile modals */
body.is-modal-open {
  overflow: hidden;
  position: fixed;
  inset-inline: 0;
  width: 100%;
}
```

The JS to toggle this class (saving/restoring scroll position) will be added in Plan 03 alongside the overlay-close behavior.

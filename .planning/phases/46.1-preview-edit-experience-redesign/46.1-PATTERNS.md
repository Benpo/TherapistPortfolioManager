## Pattern Assignments

### `assets/rich-toolbar.js` (component, event-driven) — the load-bearing file

**Analog:** itself. This is a surface rewire on top of intact mechanisms. Four concrete anchor points:

**(1) Toolbar build — where the pinned switcher + `.tb-scroll` wrapper go** (rich-toolbar.js:217-247).
Today `buildToolbar` appends groups directly to `bar`. The sketch (008-B:972) wraps formatting controls
in a scroll strip and appends the switcher AFTER it: `tb.innerHTML = '<span class="tb-scroll">' + TB_HTML
+ '</span>' + SWAP_HTML`. Map to shipped DOM: wrap the `GROUPS.forEach(...)` output in a scroll-strip
element, then append a NEW segmented switcher element as a sibling (outside the strip). The container
mousedown guard at :232 must stay verbatim (Gap-16).
```javascript
// shipped buildToolbar (rich-toolbar.js:234-245) — groups appended flat to bar today:
GROUPS.forEach(function (group, gi) {
  if (bar.childElementCount > 0) bar.appendChild(makeSeparator());
  var wrap = document.createElement("div");
  wrap.className = "rich-toolbar-group";
  group.forEach(function (spec) { wrap.appendChild(makeButton(spec)); });
  bar.appendChild(wrap);
  if (gi === 1 && _config.headings) { /* heading dropdown */ }
});
```
**Decision for planner:** the `preview` entry in `GROUPS` (rich-toolbar.js:143-145) is the OLD single
target-state toggle — it must be REMOVED from the scrollable groups and REPLACED by the two-segment
current-state switcher rendered outside `.tb-scroll`. Delete the `spec.action === "preview"` labeled-button
branch in `makeButton` (rich-toolbar.js:173-179) and `updatePreviewButton` (rich-toolbar.js:603-624)
accordingly (this whole function encodes the rejected TARGET-state semantics — Anti-Pattern in research).

**(2) Preview element — Frame treatment + in-place swap** (rich-toolbar.js:544-554, 560-587, 626-686).
Shipped `buildPreviewPane` inserts the pane `afterend` (sibling below the editor). TARGET is in-place
swap: hide the textarea, show the frame in the same box. Keep the class `.note-rendered` (the renderer
register) and keep `MdRender.render` as the SOLE innerHTML sink (rich-toolbar.js:581-583 — do NOT add a
second `innerHTML =`). The empty-state block built with `textContent` (rich-toolbar.js:564-578) survives;
only the copy changes (`toolbar.previewEmptyTitle` → "Nothing to preview yet — start typing above.").
```javascript
// shipped SOLE innerHTML sink — preserve exactly (rich-toolbar.js:580-586):
pane.classList.remove("is-empty");
if (window.MdRender && typeof window.MdRender.render === "function") {
  pane.innerHTML = window.MdRender.render(val);   // escape-first, XSS-safe — the ONLY sink
} else {
  pane.textContent = val;                          // fallback: literal, never raw innerHTML
}
```
In-place swap mechanic (sketch 009-A:626-648, canonical): `ta.classList.add('is-hidden')` +
`pv.classList.add('is-on')` + set frame `minHeight = ta.offsetHeight + 'px'`; on leave, reverse and
`ta.focus()`. Adapt `openPreview`/`closePreview` (rich-toolbar.js:626-674) to this instead of the
`afterend` pane + `is-hidden`-on-pane-only approach.

**(3) `revealPreviewPane` becomes moot — but prove before deleting** (rich-toolbar.js:646-663).
The below-fold scroll math is scoped to `.export-edit-area`. In-place swap removes the second box, so it
should be unnecessary — but the export area still `overflow-y:auto` (app.css:3400). Research Pitfall 2:
convert first, prove with WebKit set E that the frame is visible-on-open WITHOUT scroll, THEN remove.
Uses `getBoundingClientRect` (physical) correctly for vertical-only scroll — RTL-safe. Do not reintroduce
logical insets fed by physical rects.

**(4) Dispatch chokepoint — mode segments must dispatch on the bar's own field** (rich-toolbar.js:689-739).
The `preview` action already resolves `barField || _focused` without focusing (rich-toolbar.js:698-701) —
the mode segments inherit this. NEW: `Ctrl/Cmd+E` toggle (net-new binding, A1) must be bound where it
still reaches while the textarea is hidden (bar-level or document-level, guarded to active field) —
Research Pitfall 3. Formatting-click-in-preview returns-to-edit-then-applies (sketch 008-B:979-983:
`if (previewing) setMode('edit'); ta.focus(); ...apply`).

## Shared Patterns

### Focus preservation (Gap-16 guard — KEEP VERBATIM)
**Source:** rich-toolbar.js:153-159 (`bindPreserveFocus`, per-control) + rich-toolbar.js:232 (container).
**Apply to:** every toolbar control including the new switcher segments.
```javascript
bar.addEventListener("mousedown", function (ev) { ev.preventDefault(); });  // container guard, :232
```
A `mousedown` on the bar's padding/gaps must never blur the field. Tests must dispatch real `mousedown`.

### The SOLE innerHTML sink (V5 security invariant — KEEP)
**Source:** rich-toolbar.js:581-583. **Apply to:** any preview render path.
Only `MdRender.render()` output may be assigned to `innerHTML`; empty-state uses `textContent`. The rewire
must NOT introduce a second unescaped `innerHTML =`. `46.1-frame-no-surface-alt.test.js` should be
accompanied by (or extended to) a source-audit that the only innerHTML sink is MdRender output.

### RTL logical-vs-physical (project MEMORY law)
**Source:** UI-SPEC §RTL + MEMORY reference-rtl-logical-props-physical-coords.
**Apply to:** switcher pin, chip pin, all layout. Static layout = logical props (`inset-inline-start`,
flex order). Measured overlay/scroll math (`getBoundingClientRect`) = physical `left/top` — NEVER feed a
physical rect into a logical inset. Directional glyphs (indent/outdent/undo/redo) carry `.icon-flip` and
mirror via `[dir="rtl"] .icon-flip svg { transform: scaleX(-1) }`; eye/pencil/bold/italic do NOT mirror.

### Docs gate (project CLAUDE.md Definition of Done)
**Apply to:** the push. TRIGGER files needing EN help coverage: `rich-toolbar.js`, `text-edit.js`,
`export-modal.js`, `add-session.html` → topics `topic-quick-paste` (capturing-emotions) AND
`topic-single-export` + `topic-export-formats` (review-export). `app.css` + the four `i18n-*.js` are
CHANGELOG-ONLY (no `Help-Unaffected:` trailer needed). Required: one EN edit to
`assets/changelog-content-en.js` + one EN edit to `assets/help-content-en.js`. Plan a docs task.

### Comment hygiene (project MEMORY — recurring contamination)
**Apply to:** all edited files. No planning IDs (D-NN, task IDs, "Gap 15") in shipped code comments
(CONVENTIONS.md §Comments is FIXED). Reference the mechanism, not the ticket.

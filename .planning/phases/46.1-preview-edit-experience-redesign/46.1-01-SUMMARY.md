# Phase 46.1 Plan 01: Wave-0 RED Test Scaffold Summary

Four RED jsdom/source-audit tests plus an extended manual WebKit probe pin the new preview/edit experience (current-state switcher, in-place swap, Ctrl/Cmd+E, return-and-apply, pinned-outside-strip, Gap-15 Frame, sole-innerHTML-sink) BEFORE any implementation — every file fails against current shipped source and goes green only when plans 02-05 rewire the surface.

## What Was Built

- **`tests/46.1-inplace-swap.test.js`** (RTXT-04) — 9 outcome-only jsdom cases: entering preview hides the textarea (`is-hidden`) and renders the field value into `.rich-toolbar-preview-body` as real MdRender output (`**hi**` → a `<strong>`); the persistent export bar resolves preview WITHOUT focusing the editor (iOS soft-keyboard suppression); leaving preview restores + refocuses the textarea and hides the frame; `Ctrl/Cmd+E` toggles both directions (field keydown for enter, a DOCUMENT-level keydown after an explicit blur for the return path); a focusout the instant preview opens does not cascade the bar+preview closed (BLOCKER-3 guard, 0ms deferred timer flushed). The env evals `md-render.js` before `rich-toolbar.js` (index.html order) so the `<strong>` assertion can ever go green.
- **`tests/46.1-switcher-current-state.test.js`** (RTXT-04 Gap-14) — exactly two `[data-mode]` segments; the ACTIVE segment always names the CURRENT mode (kills the rejected single target-labelled toggle that lit up while in the opposite mode); distinct labels; `is-hidden` textarea corroborates the active mode.
- **`tests/46.1-format-in-preview.test.js`** (RTXT-01) — clicking a formatting button while previewing returns to Edit and applies the action in one gesture (bold adds `**`); the switcher segments are NOT descendants of the horizontally-scrollable `.rich-toolbar-scroll` strip while the formatting buttons are inside it.
- **`tests/46.1-frame-no-surface-alt.test.js`** (RTXT-04 Gap-15 + V5) — region-scoped negative audit of the base `.rich-toolbar-preview` rule (no section-header background token), positive Frame check (surface background + ghost-green border), and the sole-innerHTML-sink regression guard (the one preview `.innerHTML =` is fed by `MdRender.render` — GREEN today and must stay green).
- **`tests/webkit/46-export-step2-layout.mjs`** (extended) — retargeted `SEL.previewBtn` to `.rich-toolbar-swap-btn[data-mode="preview"]` (old `data-action="preview"` selector gone), added `SEL.editSeg`, added both to the uniqueness gate, converted the C-precondition fallback to a real `mousedown`, and added set E in-place-swap (E4 visible-on-open without scroll, E5 pinned bar, E6 editorHidden, E7 editor restored, E8 RTL precondition) across a new EN pass and an HE/RTL pass. A-D remain intact under the content-driven card model; the probe stays manual-only.

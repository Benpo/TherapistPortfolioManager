<objective>
Upgrade the preview open/close from the shipped below-editor sibling pane to an IN-PLACE SWAP (the
preview Frame replaces the editor in the same box; one thing on screen at a time) with the unified Frame
treatment (floating PREVIEW chip), and add the `Ctrl/Cmd+E` mode toggle that keeps reaching while the
textarea is hidden. This structurally eliminates the mixed-state scroll problem (Gap-14a) and the
below-fold pane (Gap-13) because there is no second box to scroll to.

Purpose: the mechanism half of the redesign — where the preview is shown and how it swaps back to edit.
Runs after plan 02 (same file) which already wired the current-state switcher to call openPreview/closePreview.
Output: rewired `buildPreviewPane` / `renderPreview` / `openPreview` / `closePreview` / `revealPreviewPane`
+ a `Ctrl/Cmd+E` binding in `assets/rich-toolbar.js`.
</objective>

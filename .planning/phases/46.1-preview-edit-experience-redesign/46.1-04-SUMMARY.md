# Phase 46.1 Plan 04: Preview Frame + Scroll Strip + Export Sizing (CSS) Summary

Styled the redesigned rich-text surfaces entirely in `assets/app.css`: the unified Frame preview treatment with a floating PREVIEW chip (the Gap-15 fix — never the section-header background), the scrollable formatting strip with a pinned green-active current-state switcher, the buttons-only dim-while-previewing rule, RTL directional-glyph mirroring, and the content-driven export card sizing (the Gap-10 fix — `block-size: auto` + a viewport cap replacing the shipped hard 50dvh, with a 280px editor floor). All values reuse shipped `tokens.css` tokens on the 4px grid — no new palette.

## What was built

- **Frame treatment (Task 1).** `.rich-toolbar-preview` is now the Frame wrapper: `position: relative`, `background: var(--color-surface)`, `border: 1.5px solid var(--color-green-border-ghost)`, `border-radius: 12px`, `padding: 20px 16px 14px`. The Gap-15 offender (`background: var(--color-surface-alt)`) is gone. A `[data-theme="dark"] .rich-toolbar-preview` rule falls the border back to `--color-border`.
- **Floating chip + body.** `.rich-toolbar-preview-chip` is an absolute, logical-inset (`inset-inline-start: 14px`) uppercase pill using `--color-surface-secondary-btn` / `--color-primary-dark`. `.rich-toolbar-preview-body` carries only `background: transparent` (it wears `.note-rendered` for text; the wrapper's 20px top padding clears the chip).
- **Export Frame flex-fill.** `.export-card.is-editor-step .rich-toolbar-preview { flex: 1 1 auto; min-block-size: 0; overflow-y: auto; }` — placed after the base rule so the source-audit still reads the Frame treatment.
- **Scroll strip + pinned switcher (Task 2).** `.rich-toolbar` is now a non-scrolling flex row (`overflow: visible`); horizontal scroll moved to the inner `.rich-toolbar-scroll` (`flex: 1 1 auto; overflow-x: auto`). `.rich-toolbar-swap` pins by flex order; `.rich-toolbar-swap-btn.is-active` is the green fill (`--color-primary` + `--color-text-inverse`).
- **Dim + RTL.** `.rich-toolbar.is-preview .rich-toolbar-scroll .rich-toolbar-btn { opacity: 0.4; }` dims formatting buttons only (heading trigger included, no double-dim); the switcher outside the strip is untouched. `html[dir="rtl"] .rich-toolbar .icon-flip svg { transform: scaleX(-1); }` mirrors only the directional glyphs.
- **Content-driven export sizing.** `.export-editor` floor 240 → 280px. `.export-card.is-editor-step` rewritten from the hard `block-size: 50dvh` + `min(640px,90dvh)` floor to `block-size: auto` + `max-block-size: 94vh/94dvh`; maximize caps at `92vh/92dvh` with `block-size: auto`. The sticky/`flex-shrink:0` export bar (untouched) and the `@media (max-width:768px)` phone full-screen takeover (untouched) survive.
- **Cleanup.** `.export-info-note { display: none }` added inside the existing `@media (max-width:768px)` block. Dead `.rich-toolbar-btn--labeled` / `.rich-toolbar-btn-label` rules removed (their emitter was deleted by plan 02), and the labeled line in `@media (max-width:640px)` replaced with `.rich-toolbar-swap-btn { min-height: 44px; }`.

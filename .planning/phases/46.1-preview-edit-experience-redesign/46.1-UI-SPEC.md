# Phase 46.1 — UI Design Contract

> Visual and interaction contract for the Preview & Edit experience across BOTH rich-text
> surfaces: the 7 session note fields (add/edit session) AND the export Step-2 flow.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Source of truth:** the Ben-ratified sketch winners — **008 variant B** (export in-place
> swap) and **009 variant A** (note-field in-place swap), both with the **Frame** preview
> treatment and a **pinned** Edit/Preview switcher (Ben, 2026-07-17/18). This document
> translates those two mockups into an executable contract. Where the mockup markup/CSS and
> this document agree, the mockup is the canonical visual reference:
> `.planning/sketches/008-export-preview-edit-model/index.html` and
> `.planning/sketches/009-note-field-preview/index.html`.
>
> **This is a re-presentation phase** (RTXT-01, RTXT-04, RTXT-05 already shipped). The
> engineering mechanisms underneath — the 46-15 sizing floor, the sticky/persistent bar, the
> dispatch-without-focus fix (46-17), the below-fold preview reveal (46-17) — survive as
> mechanism; only the *surfaces* they present are being redesigned. Several checker
> dimensions carry accepted-exception notes because this reuses the shipped token system
> rather than inventing a greenfield one (see Checker Sign-Off).

## Spacing Scale

Declared values (all multiples of 4 — already tokenized in `tokens.css`):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| `--space-xs` | 4px | Toolbar button gaps, bar padding, chip inner gap |
| `--space-sm` | 8px | Toolbar margin-block-end, control-strip gaps, split-grid gap (mobile) |
| `--space-md` | 16px | Field spacing, preview-frame horizontal padding, form-card padding (mobile) |
| `--space-lg` | 24px | Form-card padding (desktop), section padding |
| `--space-xl` | 32px | Modal/layout gaps |
| `--space-2xl` | 48px | Major section breaks |
| `--space-3xl` | 64px | Page-level spacing |

Radii (shipped conventions, reused as-is): `--radius-sm` 8px (toolbar buttons), `--radius-md`
12px (editor, preview-frame, toolbar, note-field), `--radius-lg` 18px (form-card / modal-card),
`--radius-full` 9999px (mode switcher, preview-chip).

**Exceptions:**
- **44px minimum tap target** on phone widths (`--tap-target-min: 44px`) — toolbar buttons grow
  from 36px→44px under `body.vp-phone`. This is an accessibility floor, not a scale violation.
- **Editor min-height floor: 280px** on the export editor (`.step2-editor { min-height: 280px }`)
  — the 46-15 sizing-floor mechanism. Not a spacing token; it is the load-bearing sizing
  guarantee (see Interaction Contract §Sizing). 96px is the equivalent floor for the
  autogrowing note fields.

## Typography

Four roles for these surfaces. Line-heights are prescriptive.

| Role | Size | Weight | Line Height | Where |
|------|------|--------|-------------|-------|
| Editor / body text | 16px (`1rem`) | 400 | 1.55 | `.step2-editor`, `.note-field`, rendered `<p>` |
| Field label | 15px (`0.95rem`) | 600 | 1.4 | `.field > label`, export step lead copy |
| Rendered heading (preview/PDF register) | H1 17px · H2 16px · H3 15px | 600 | 1.35 | `.note-rendered h1/h2/h3` — **copied verbatim from shipped `app.css`; do not alter** |
| Modal title (export) | 20px (`1.25rem`) | 700 | 1.2 | `.modal-title` "Export Session" |
| Preview chip | ~11.5px (`0.72rem`) | 600 | — | `.preview-chip` "PREVIEW", uppercase, `letter-spacing: 0.04em` |
| Mode switcher label | ~13.6px (`0.85rem`) | 400 base / 600 when active | 1 | `.swap button` — active segment goes 600 |

**Accepted exception (weights):** the surfaces use **three** shipped weights — 400 (body/editor),
600 (labels, rendered headings, chip, active mode segment), 700 (modal title). The greenfield
"exactly 2 weights" rule does not apply: this is a reuse-only re-presentation of the shipped
Rubik register (`app.css` `.note-rendered` already ships 600 headings; `tokens.css` exposes
400/700 as `--weight-regular`/`--weight-bold` while 600 is used semantically for emphasis-lite).
Per the recorded gsd-ui-checker greenfield false-positive guidance, grep the live source and
approve with this note rather than collapsing weights.

## Color

60 / 30 / 10 split, drawn entirely from `tokens.css` (light values shown; dark-mode values in
`[data-theme="dark"]` are the shipped teal-grey palette and must both be honored).

| Role | Token (light / dark) | Usage |
|------|----------------------|-------|
| Dominant (60%) | `--color-background` `#fdf8f0` / `#1a2020` | Page background, modal backdrop area, form page |
| Secondary (30%) | `--color-surface` `#ffffff` / `#202828` | Cards, editor, **preview-frame**, toolbar, note-field |
| Accent (10%) | `--color-primary` `#2d6a4f` / `#5aacac` | See reserved-for list below |
| Destructive | `--color-danger` `#ea4b4b` (light-only by design) | "Discard" button in the discard-edits confirm dialog **only** |

**Accent (`--color-primary`) reserved for — explicit list, never "all interactive elements":**
1. The **active** segment of the Edit/Preview mode switcher (green fill + `--color-text-inverse`
   text) — the single strongest signal of "where you are". Current-state, not target-state.
2. The primary CTA button (`Continue` / `Export`) — `.btn-primary`.
3. Editor focus border (`.step2-editor:focus` / `.note-field:focus { border-color: primary }`).
4. Preview-chip text (`--color-primary-dark`) and the Frame preview border tint
   (`--color-green-border-ghost`, a 25%-alpha primary).
5. Active toolbar button state (`.tb-btn.is-active` — e.g. the current heading level), using the
   soft `--color-surface-secondary-btn` fill with `--color-primary` glyph.

**LOAD-BEARING COLOR RULE (Gap 15 — the reason this phase exists):** the preview treatment must
**NEVER** use `--color-surface-alt`. That token is the accordion/section-header background
("Limiting Beliefs"), and reusing it is exactly what made the old preview read as another
category header. The ratified **Frame** treatment is: `background: --color-surface` +
`border: 1.5px solid --color-green-border-ghost` (dark mode: `--color-border`) + the floating
`.preview-chip`. Paper and Tint treatments were explored and **rejected** — Frame is the single
unified "this is preview" language on both surfaces.

## Copywriting Contract

Both surfaces. EN is the corpus of record; HE/DE/CS follow in the locale pass (this app is
trilingual+CS with full RTL — every string below needs the four-locale treatment during
implementation, EN authored here).

| Element | Copy |
|---------|------|
| Export primary CTA (steps 1→2) | **Continue** |
| Export primary CTA (step 3) | **Export** |
| Export secondary / back | **Back** (and, where discard applies, the confirm buttons below) |
| Mode switcher — edit segment | **Edit** (pencil icon) |
| Mode switcher — preview segment | **Preview** (eye icon) |
| Preview chip (floats on the frame) | **PREVIEW** (uppercase) |
| Export ephemeral-edit info note | **Edits here apply to this export only.** Restructure freely — nothing is saved back to the session. |
| Empty-state (note-field preview, no text yet) | **Nothing to preview yet — start typing above.** |
| Export step labels (3-step spine) | **Choose · Edit · Export** |
| Destructive confirmation — title | **Discard your export edits?** |
| Destructive confirmation — body | These changes apply to this export only and won't be saved. |
| Destructive confirmation — buttons | **Keep editing** (secondary) · **Discard** (danger) |

**Mode-switcher semantics are copy AND state (the Gap-14 fix):** the switcher shows the CURRENT
mode, not the target. While editing, the **Edit** segment is green/filled and the **Preview**
segment is inert; entering preview flips the green to the **Preview** segment. Never a single
target-labelled button whose active styling contradicts its label (the rejected old design read
"Edit"-labelled-and-green while actually in preview).

**Error state:** these surfaces have no data/network error states (rendering is local and
synchronous). The empty-preview copy above is the only "nothing to show" state. No error-toast
copy is owned by this phase.

**Destructive actions in scope:** exactly one — discarding export edits (ephemeral, this-export-only)
via the confirm dialog above. The 7 note fields have **no** destructive action here (their text
saves with the session form as normal; leaving preview never discards anything).

## Interaction Contract

> The heart of this phase. These answer Ben's verbatim questions: *"which buttons to show, where
> the preview is shown and how to go back to edit, how much screen space to give each part, how
> the overall process looks."* All items are **ratified/LOCKED** unless marked otherwise.

### The mode model (both surfaces) — LOCKED
- **In-place swap.** Preview fully **replaces** the editor in the same box; one thing on screen
  at a time. There is never a stacked editor+preview in one scroll container (the rejected
  concept). Note fields: preview takes the textarea's exact height. Export: preview takes the
  editor's box within the fixed step-2 area.
- **Current-state switcher**, pinned. The Edit/Preview segmented control lives at the **end of
  the toolbar, OUTSIDE the horizontally-scrollable control strip** (`.tb-scroll`), so on phone
  widths the formatting buttons scroll beneath it while **Preview stays always visible** without
  scrolling the bar. Active segment = green fill (`--color-primary` + inverse text).
- **Formatting-in-preview:** formatting buttons remain visible but **dim to `opacity: 0.4`**
  (`.rt-toolbar.is-muted [data-act]`) while previewing; clicking one **returns to Edit and applies
  the action** in a single gesture. The switcher segments themselves never dim (`.never-mute`).
- **Keyboard:** `Ctrl/Cmd+E` toggles Edit ↔ Preview on both surfaces. `Ctrl/Cmd+B` bold,
  `Ctrl/Cmd+I` italic, `Tab`/`Shift+Tab` indent/outdent in lists, `Enter` continues lists.

### Toolbar button set (which buttons to show) — LOCKED (shipped set, reused)
Left-to-right inside the scrollable strip: **Text-style** dropdown (cycles Text → H1 → H2 → H3),
sep, **Bold**, **Italic**, sep, **Bullet list**, **Numbered list**, **Indent**, **Outdent**, sep,
**Undo**, **Redo**. Then, pinned outside the strip: the **Edit / Preview** switcher. Buttons are
36×36px (44×44px on phone). Directional glyphs (indent, outdent, undo, redo) carry `.icon-flip`
and mirror under `[dir="rtl"]`.

### Export surface (Step 2) — the 3-step spine, LOCKED
- **Spine: Choose → Edit → Export.** Preview is a **mode within the Edit step**, not a separate
  step (variant C's "Preview & Export stage" was rejected in favor of B's single-surface clarity).
- The export toolbar is **persistent and always visible** (ratified invariant, Gap 2/11): it
  renders permanently above the editor, `flex-shrink: 0` so it can never compress/clip, and is
  `position: sticky; inset-block-start: 0` (or docked outside the scroll container) so it never
  scrolls away on long documents. Verify at DEFAULT (non-maximized) size on a real MacBook
  viewport, not only maximized.
- **Sizing (the 46-15 floor):** the **edit surface**, not the card, is the sizing floor. Card
  height is **content-driven with a max-height cap** (`height: auto; max-height: 94%`); the
  editor's `min-height: 280px` grows the card rather than collapsing to a sliver. Maximize →
  ~92%. Phone → full-screen takeover (D-17). Never hard-size the card to `50dvh` (the Gap-10
  root cause).
- **Persistent-bar dispatch (Gap 12 mechanism, survives):** every toolbar control must act on the
  bar's own field even before the editor is focused (resolve `barField || _focused`, and focus as
  part of the action). A visible always-on bar invites first-click use.
- **Preview reveal (Gap 13 mechanism, survives):** entering preview must bring the preview box
  into view (it replaces the editor in-place, so with in-place swap this is inherent — but on any
  residual scroll offset, scroll the pane's top below the pinned bar). Must not disturb the sticky
  bar, undo, or caret state; RTL-safe.
- **Ephemeral-edit info note** shown at the top of Step 2 (`--color-info-bg`/`--color-info-text`);
  hidden on phone to preserve editing pixels.

### Note-field surface (7 fields on add/edit session) — LOCKED
- **Focus-attached toolbar:** the single shared bar docks directly above the focused field on
  focus (`is-docked`, a 0.15s fade/translate-in), and undocks on click outside any field/bar.
  This differs from the export's persistent bar by design (note fields keep focus-attached).
- **In-place swap** replaces the textarea with the preview frame at the same height; leaving
  preview restores the textarea and refocuses it.
- **Autogrow:** note fields autogrow to content (min 96px); preview matches the field's height.
- **Empty state:** if the field has no text, preview shows the empty-state copy inside the frame
  rather than an empty box.
- **Gap-16 guard (shipped, keep):** `mousedown + preventDefault` on the bar **container** (not
  just per-control) so a click on the bar's padding/gaps never blurs the field and collapses it.

### RTL / i18n contract (applies to every item above)
- **Logical properties only** for layout: `inset-inline-*`, `margin-block-*`, `padding-inline-*`,
  `border-inline-*`. The chip pins with `inset-inline-start`; the switcher pins at the bar's
  logical end.
- **Directional icons mirror** via `[dir="rtl"] .icon-flip svg { transform: scaleX(-1) }`
  (indent/outdent/undo/redo). Semantic glyphs (bold, italic, eye, pencil, list bullets) do NOT
  mirror.
- **Physical coords for any measured overlay math** (per the recorded RTL logical-vs-physical
  gotcha) — if any reveal/scroll math reads `getBoundingClientRect`, use physical left/top, never
  feed physical rects into logical insets.
- All new/changed strings authored EN here; HE/DE/CS parity required in the implementation locale
  pass (EN is the corpus of record per the docs gate).

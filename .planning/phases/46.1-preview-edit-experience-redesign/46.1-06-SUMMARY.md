# Phase 46.1 Plan 06: Docs — Changelog & Help for Preview/Edit Redesign Summary

Documented the redesigned preview/edit experience in the EN corpus of record so the phase's push passes the project Definition of Done / docs hard-gate: extended the unreleased v1.4.0 changelog entry and updated three affected help topics to describe the current-state Edit/Preview switch, the in-place framed PREVIEW view on both note fields and the export editor, and the Ctrl/Cmd+E toggle.

## What Was Built

**Task 1 — EN changelog (assets/changelog-content-en.js, commit 3eb37f0):**
Reworded the v1.4.0 `new` preview bullet. The prior copy described the OLD next-state button model ("reads Preview while you are writing and Edit while you are previewing") — which the redesign replaced. The new copy names both surfaces (note fields + export editor), the in-place framed PREVIEW look (one clean view at a time), the current-state Edit / Preview switch, click-any-formatting-button to return, and the Ctrl/Cmd+E toggle. No version bump (1.4.0 is preprod-only per Ben's decision — the extended entry ships to users together with the redesign).

**Task 2 — EN help topics (assets/help-content-en.js, commit 521a8e4):**
- `topic-quick-paste` (capturing-emotions): reworded the preview note from the old next-state model to the note-field Edit/Preview switch — in-place framed preview, active-side-highlighted current-state, three ways to return to writing (tap Edit, tap any formatting button, Ctrl/Cmd+E), plus a reassurance that snippets and field autogrow work as before.
- `topic-single-export` (review-export): described the export Step-2 always-visible toolbar (never scrolls away) and its Edit/Preview switch — in-place preview, Ctrl/Cmd+E, this-export-only edits.
- `topic-export-formats` (review-export): added a sentence about previewing the finished framed document before choosing a format.

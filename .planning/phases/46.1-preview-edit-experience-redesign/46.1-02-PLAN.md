<objective>
Replace the rejected single target-state preview toggle with a two-segment CURRENT-STATE Edit/Preview
switcher pinned outside a horizontally-scrollable formatting strip, on BOTH rich-text surfaces (the 7
note fields AND the persistent export bar), by editing the ONE `buildToolbar()` they share. This is the
Gap-14 fix: the switcher shows where you ARE (active segment = current mode), never a green label that
contradicts the mode.

Purpose: the interaction-model half of the redesign — which buttons to show and how the mode reads at a
glance. The in-place-swap mechanics land in plan 03 (same file, next wave); this plan wires a working
switcher over the existing preview open/close so an intermediate build already reads correctly.
Output: rewired `buildToolbar` / `makeButton` / `_dispatch` in `assets/rich-toolbar.js` with a
`setMode(mode)` current-state controller.
</objective>

<objective>
Solve the phase's single hardest architectural problem (D-03) and its visual constraint (D-02): make the PDF pipeline distinguish note-content headings from document-structure headings. Today `buildSessionMarkdown()` concatenates note field values into the SAME flat markdown string it uses to build the `#`/`##` document skeleton, so a user-typed `## Progress` inside a note is indistinguishable from a document `## Section` — it would wrongly get the branded leaf-diamond+green-rule chrome AND corrupt the severity-block page-break count at `pdf-export.js:1313`. This plan tags note-body blocks as a distinct category, routes note headings to a subordinate chrome-free render (UI-SPEC §C: 12/11/10.5pt bold, no chrome), and excludes them from the section count. It also locks the RTXT-08 verbatim copy/share pass-through with a falsifiable test.

Purpose: Notes must render as notes, not forge document sections. Sequenced after Plan 02 (both touch pdf-export.js) so the export surface is edited one hand at a time.

Output: A note-aware PDF pipeline (D-02/D-03) + a byte-for-byte copy/share lock (RTXT-08).
</objective>

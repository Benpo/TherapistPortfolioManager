<objective>
Stand up the primary NEW read surface: a rendered-HTML overlay in the add-session read mode so saved formatted notes display as styled text instead of literal `**`/`-` tokens (RTXT-06). Wire the three compact surfaces (sessions table cell, overview comments line, add-session spotlight quote) to strip markdown to plain text (D-06), add the `.note-rendered` CSS register (UI-SPEC §B), and extend the render-hardening test locks to cover the new, narrow innerHTML exception.

Purpose: Reading a session is the everyday surface. Today `setReadMode(true)` only sets `textarea.readOnly` — a saved `**bold**` shows literally. This plan makes notes look right on read, using MdRender (Plan 01) as the single escape-first innerHTML path, while keeping every other surface textContent-only.

Output: Read-mode formatted rendering + stripped compact surfaces + scoped CSS + extended hardening locks.
</objective>

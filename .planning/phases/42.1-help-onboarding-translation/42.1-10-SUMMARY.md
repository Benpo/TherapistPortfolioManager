## Issues Encountered

- The new chevron render assertion initially read `matrix(0,1,-1,0,0,0)` (rotate90) for every state because the `.chev` `transition: transform .18s` makes a synchronous class-toggle + immediate `getComputedStyle` return the mid-transition matrix. Resolved by setting `style.transition='none'` and forcing a reflow before each measurement — then RTL-collapsed correctly reads `matrix(-1,0,0,-1,0,0)` and LTR-collapsed reads `none`.
- Confirmed `changelog.html` has NO collapse/chevron component (grep found none), so the chevron fix is help-center-only — noted here per the orchestrator's request.

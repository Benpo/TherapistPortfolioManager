# Phase 42.1 Plan 08: Wire Loaders, Offline Delivery, RTL Rendering & Chrome i18n Summary

Locale-aware help + What's-New rendering with per-section/per-entry EN fallback, the RTL css re-key off an EN-fallback marker, changelog/help locale `<script>` siblings on all nine popup pages with 6-file precache, and full v1.3 help-chrome/AT-aria/tour-screen-name i18n wiring — so translated help, changelog, and popup content reaches users natively, RTL-correct, online and offline.

## What Was Built

**Task 1 — help.js `localeSections()` + RTL css re-key (BLOCKER 2).** Added a `lang()` helper and a `localeSections()` merge modeled on `changelog.js` `localeEntries()` but keyed on section `id`: EN is the canonical ordered base, each section swapped for its same-id locale counterpart when present, else EN. `render()` now sources from it. The merge rebuilds module-state `_fallbackIds` (section id → EN-fallback); `buildCard()` stamps `.is-en-fallback` on fallback cards. `help.css`'s interim blanket `[dir=rtl]→LTR` override on `.card-body` / `.card-head h2` / `.page-head > p` was replaced with a narrow rule scoped to `.help-card.is-en-fallback` bodies/headings only — so translated prose renders RTL while a per-section EN fallback still reads LTR. The now-translated page-head intro is no longer force-flipped.

**Task 2 — whats-new.js locale-aware `entries()` (Pitfall 1).** Rewrote `entries()` to resolve `portfolioLang` (→ `I18N_DEFAULT` → `en`), read `CHANGELOG_CONTENT_<LANG>`, and map each EN entry to its same-version localized counterpart with per-entry EN fallback. `entryFor(v)` inherits the fix unchanged; render path stays `textContent`.

**Task 3 — locale `<script>` siblings + precache (BLOCKER 1).** Added `changelog-content-{he,de,cs}.js` after `changelog-content-en.js` on all nine whats-new.js pages (index, sessions, add-client, add-session, settings, reporting, report, help, changelog), added `help-content-{he,de,cs}.js` on help.html, and added all 6 files to `sw.js` `PRECACHE_URLS`. `CACHE_NAME` untouched (auto-rolls via `INTEGRITY_TOKEN`).

**Task 4 — chrome + tour screen-name i18n (BLOCKER 3 + WARNING 3).** Removed help.js's hardcoded `LABELS` object; all render sites (startHere, sessionLoop, technicalBits rail label + tech-band aria, techBand label, spine) and the rail-toggle "Show topics" aria resolve through `help.chrome.*` via `t()`. `render()` now sets the `#rail`, `#jumpNav`, and search-input aria-labels from `help.chrome.*` so they flip live on `app:language`. help.html carries `data-i18n` for intro, contact title/body, jump-to-section (caret span preserved as a sibling), and report-hint. tour.js resolves the fallback `{screen}` name through a `SCREEN_KEYS` → `help.tour.screen.*` lookup with a literal fallback.

# Phase 42.1 Plan 06: Changelog Content Locale Siblings Summary

Authored the three changelog locale siblings — `assets/changelog-content-{he,de,cs}.js` — each registering `window.CHANGELOG_CONTENT_<LOC>` with the full 4-entry native history (v1.3, v1.2, v1.1, v1.0 origin), structurally byte-identical to EN, so the changelog page and What's-New popup will read natively in every locale once Plan 08 wires the loaders.

## What was built

- **Task 1 — Hebrew** (`changelog-content-he.js`): 4-entry array, RTL house register, Heart-Wall → חומת הלב, client → לקוח, session → מפגש. Months localized to יולי/יוני/מאי 2026. Gender-neutral impersonal phrasing (פותחים) per project convention.
- **Task 2 — German** (`changelog-content-de.js`): Sie form, shipped i18n-de.js vocab (Klient, Sitzung, Sitzungsart, Textbausteine), Heart-Wall kept Latin as in the app. Months Juli/Juni/Mai 2026.
- **Task 3 — Czech** (`changelog-content-cs.js`): formal register, shipped i18n-cs.js vocab (klient, sezení, typ sezení, úryvky), Heart-Wall kept Latin. Months červenec/červen/květen 2026.

Each file: IIFE registering only its own `CHANGELOG_CONTENT_<LOC>` global (verified: loading a locale file in a fresh sandbox leaves `CHANGELOG_CONTENT_EN` undefined). `version`/`anchor`/`origin`/category-key-set + order are byte-identical to EN; the v1.0 entry stays origin-only. No emoji, no control/bidi chars, no clinical terms.

## Deviations from Plan

### Auto-fixed / constraint-resolved

**1. [Rule 3 - Blocking constraint] HE "Session Format" term**
- **Found during:** Task 1
- **Issue:** The plan says to follow shipped he vocab for Session Format, but the shipped label is `filter.sessionFormat` = "אופן הטיפול", which contains טיפול — a term the integrity gate's HE `FORBIDDEN_TERMS` regex (`/טיפול/`) rejects. Using it verbatim would fail the gate.
- **Fix:** Rendered Session Format as סוג המפגש / אופן המפגש and the built-in session formats as סוגי מפגשים — natural Hebrew that carries the same meaning without the forbidden root.
- **Files:** assets/changelog-content-he.js
- **Commit:** 3225d02

No other deviations — structure, ordering, versions, anchors, and category keys are byte-identical to EN as required.

<objective>
Run the native-speaker register gate — the locked pipeline step between agent translation and Sapir's human read. For each locale, review the WHOLE v1.3 corpus together (help body, changelog, welcome, all 42 tour keys including the 31 that were gate-only in Plan 07, AND the full v1.3 chrome key set: changelog.* + whatsNew.* + onboard.install.*/onboard.mobileHint.* + the new help.chrome.*/help.tour.screen.* keys), judging register consistency, terminology, and copy discipline as a native speaker would, and apply refinements. Register consistency is a whole-corpus property, so this pass is deliberately separate from the per-file drafting plans. This is the ONLY human backstop DE/CS get — so its scope must be the whole v1.3 surface, not just help/tour (WARNING 1).

Purpose: L10N-01 — the locked pipeline requires native-speaker agent gates (Sonnet) per locale before ship; CS ships on this gate alone (accepted risk), HE additionally goes to Sapir (Plan 10).
Output: register-gated, refined he/de/cs corpora (content + full chrome) with all machine gates green.
</objective>

<objective>
Run the native-speaker register gate — the locked pipeline step between agent translation and Sapir's human read. For each locale, review the WHOLE v1.3 corpus together (help body, changelog, welcome, and all 42 tour keys including the 31 that were gate-only in Plan 07), judging register consistency, terminology, and copy discipline as a native speaker would, and apply refinements. Register consistency is a whole-corpus property, so this pass is deliberately separate from the per-file drafting plans.

Purpose: L10N-01 — the locked pipeline requires native-speaker agent gates (Sonnet) per locale before ship; CS ships on this gate alone (accepted risk), HE additionally goes to Sapir (Plan 10).
Output: register-gated, refined he/de/cs corpora with all machine gates green.
</objective>

Do NOT add CS external human review (accepted risk — this agent gate is the CS ceiling).

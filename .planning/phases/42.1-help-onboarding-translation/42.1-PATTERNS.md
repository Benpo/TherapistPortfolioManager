## Shared Patterns

### Shared Pattern 1 ‚Äî Locale-merge with per-entry EN fallback (THE canonical merge, D-16)

**Source:** `assets/changelog.js:59-67` (VERBATIM ‚Äî already exists, tested, green)
**Apply to:** `help.js` (new `localeSections()`), `whats-new.js` (new locale-aware `entries()`)

```javascript
// ‚îÄ‚îÄ locale-merged entries (D-16 EN-fallback) ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ
function localeEntries() {
  var en = window.CHANGELOG_CONTENT_EN || [];
  var loc = window["CHANGELOG_CONTENT_" + lang().toUpperCase()];
  if (!loc || !Array.isArray(loc)) return en;
  return en.map(function (base) {
    var match = loc.filter(function (e) { return e && e.version === base.version; })[0];
    return match || base;
  });
}
```

`lang()` in changelog.js (lines 32-36) ‚Äî copy this shape into help.js if not already present:

```javascript
function lang() {
  return (window.App && typeof App.getLanguage === "function")
    ? String(App.getLanguage() || "en")
    : "en";
}
```

**Merge key differs per surface:** changelog/whats-new key on `e.version`; help keys on section `s.id`. EN stays the canonical, always-complete ordering; each element is swapped for its same-key localized counterpart when present, else stays EN.

### Shared Pattern 2 ‚Äî `{ui:key}` tokens are NOT translated (D-23)

**Source:** `assets/help.js:84-110` (resolver ‚Äî unchanged this phase); `help-content-en.js:36-41` (contract comment)
**Apply to:** all `help-content-{he,de,cs}.js` authors and the per-locale integrity test

The resolver already flips labels to the live locale at render time:

```javascript
// help.js:84-89
function interpolateUiLabels(text) {
  if (text == null) return "";
  return String(text).replace(/\{ui:([^}]+)\}/g, function (_m, key) {
    return t(key.trim());
  });
}
```

**Translator rule:** translate only the surrounding prose; leave every `{ui:...}` token **byte-for-byte identical to EN**. The token auto-resolves to the correct locale label. The per-locale test MUST re-run the "every `{ui:key}` resolves against `window.I18N.<loc>`" check (mirror `39-help-integrity.test.js:108-125`).

### Shared Pattern 3 ‚Äî textContent-only render path (XSS boundary ‚Äî DO NOT break)

**Source:** `help.js:22-24`, `changelog.js:18-20`, `whats-new.js:139-148`
**Apply to:** nothing new ‚Äî the render path is already safe. Rule: translated content is **data**; never introduce `innerHTML` for RTL, formatting, or any reason. Bodies are `textContent`-injected, so the literal `\n` is the ONLY paragraph-break mechanism (see Pitfall in `help.tour.step.help.body`).

### Shared Pattern 4 ‚Äî vm-sandbox test loader (fs + vm.createContext)

**Source:** `39-help-integrity.test.js:33-52` and `41-tour-i18n-parity.test.js:33-53` (identical idiom)
**Apply to:** both new per-locale test files

```javascript
const fs = require('fs');
const path = require('path');
const vm = require('vm');

const sandbox = { window: {}, console: { log() {}, warn() {}, error() {} } };
sandbox.window.I18N = {};
sandbox.window.QUOTES = {};
vm.createContext(sandbox);

const ASSETS_DIR = path.join(__dirname, '..', 'assets');
const FILES = ['i18n-en.js', 'help-content-en.js']; // + the locale files being validated
for (const f of FILES) {
  const src = fs.readFileSync(path.join(ASSETS_DIR, f), 'utf8');
  try {
    vm.runInContext(src, sandbox, { filename: 'assets/' + f });
  } catch (err) {
    console.error('FATAL: assets/' + f + ' failed to load in vm sandbox.');
    console.error('       ' + (err && err.stack || err.message || err));
    process.exit(1);
  }
}
```

**Test contract (all files):** custom `test(name, fn)` harness (39-help-integrity:96-106), `console.log` PASS/FAIL, `process.exit(failed === 0 ? 0 : 1)`. `tests/run-all.js:46` auto-discovers any top-level `tests/*.test.js` ‚Äî new files join the suite with zero registration.

### `assets/help-content-{he,de,cs}.js` (content-data, transform)

**Analog:** `assets/help-content-en.js` (structure) + `assets/changelog-content-en.js` (locale-sibling convention)

**IIFE + global registration** (`help-content-en.js:47-50`):
```javascript
(function () {
  "use strict";
  var SECTIONS = [
    // ‚Ä¶12 sections‚Ä¶
  ];
  window.HELP_CONTENT_EN = SECTIONS;  // locale file registers HELP_CONTENT_HE / _DE / _CS
})();
```

**Section/topic schema** (`help-content-en.js:14-45` header) ‚Äî structurally identical per locale:
- `section`: `id` (kebab, deep-link target), `title`, `group` ('session-loop'|'technical'), `featured` (boolean, exactly ONE), `topics[]`.
- `topic`: `id`, `title`, `priority` (1|2|3), `covers[]` (repo paths, non-empty), `body[]`.
- body nodes: `{type:'p',text}`, `{type:'note',text}`, `{type:'steps',items:[...]}`, `{type:'glyph',name}`.

**TRANSLATE only:** section `title`, topic `title`, and body `text` / `steps[]` items.
**NEVER touch (must be identical to EN):** section `id`, topic `id`, `group`, `featured`, `priority`, `covers[]`, `{ui:key}` tokens, node `type`, glyph `name`, array order.
**`window.HELP_DEEPLINKS`** lives ONLY in the EN file ‚Äî do NOT redefine it in locale siblings (the loader reads it once). Locale files register ONLY `window.HELP_CONTENT_<LANG>`.

### `assets/help.js` (loader, transform ‚Äî MODIFIED)

**Analog:** `changelog.js:59-67` (Shared Pattern 1)

**Current EN-hardwired read** (`help.js:349-351`) ‚Äî the exact lines to replace:
```javascript
function render() {
  var sections = window.HELP_CONTENT_EN;
  if (!Array.isArray(sections)) return;
```

**Change:** add a `localeSections()` helper (keyed on section `id`) and source `render()` from it:
```javascript
function localeSections() {
  var en = window.HELP_CONTENT_EN || [];
  var lang = (window.App && App.getLanguage) ? String(App.getLanguage() || "en") : "en";
  var loc = window["HELP_CONTENT_" + lang.toUpperCase()];
  if (!Array.isArray(loc)) return en;                    // whole locale missing ‚Üí all EN
  return en.map(function (base) {
    var match = loc.filter(function (s) { return s && s.id === base.id; })[0];
    return match || base;                                // per-section EN fallback
  });
}
```

**Live re-render already wired** ‚Äî no logic change needed: `help.js:455-463` listens for `app:language` and re-runs `render()` (re-applying the active search). Once `render()` sources from `localeSections()`, the locale switch works end-to-end.

function entries() {
  var lang = (lsGet('portfolioLang') || window.I18N_DEFAULT || 'en').toUpperCase();

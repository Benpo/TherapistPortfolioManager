# Phase 42.1 Plan 02: RED-First Integrity Gate Extensions Summary

Extended six existing test gates RED-first (one RED-safe) so the phase's translation and wiring plans have falsifiable targets for the highest-risk mechanical failures: the dropped `\n\n` paragraph break, the empty `help.welcome.subtitle2` stubs, the 17 new v1.3 chrome/screen keys, `{V}`/`{X.Y}` token loss, the six missing offline-precache entries, the What's-New popup locale asymmetry, and the missing popup-locale `<script>` tags on the nine chrome pages.

## What Was Built

- **Task 1 — `\n\n` break (tests/41-tour-i18n-parity.test.js):** Added `requireToken('help.tour.step.help.body', '\n\n')` to the existing placeholder-tokens test. EN passes (its value carries the break); he/de/cs go RED until Plan 07 re-derives them. The `{screen}/{n}/{total}` guards are intact.
- **Task 2 — subtitle2 + 17 keys (tests/40-i18n-parity.test.js):** Added `help.welcome.subtitle2` to `NEW_KEYS` and a new `CHROME_KEYS_421` array (13 `help.chrome.*` + 4 `help.tour.screen.*`), folded into a combined `GUARDED_KEYS` set that drives the presence/non-empty, parity, and no-emoji loops. Test names made count-agnostic. RED now: 71 issues (17 keys missing in every locale + subtitle2 empty in he/de/cs). The 42-key tour set in file 41 is untouched.
- **Task 3 — precache + popup gates (tests/39-help-precache.test.js, tests/42-precache.test.js, tests/42-whats-new-gating.test.js):**
  - 39: require `help-content-{he,de,cs}.js` in PRECACHE_URLS.
  - 42-precache: require `changelog-content-{he,de,cs}.js` in PRECACHE_URLS AND assert they are absent from PRECACHE_HTML (region isolation preserved).
  - 42-whats-new T-42-V4: seed `CHANGELOG_CONTENT_HE` + `portfolioLang='he'`, drive `show()`, assert the Hebrew highlight renders and the English one does not — RED until Plan 08 makes `entries()` locale-aware.
  - 42-whats-new T-42-V5: static gate reading all nine whats-new.js pages, requiring the three changelog-content locale siblings on each, with a drift guard (`readdirSync` filter == audited nine). RED now (27 gaps).
- **Task 4 — token survival (tests/42-i18n-parity.test.js):** Added a `requireToken`-style test asserting `{V}` in `changelog.entry.version` and `{X.Y}` in `whatsNew.title` across all four locales. RED-safe — PASSES today; guards future drift and the hardcoded-EN fallback.

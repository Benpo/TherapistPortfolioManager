### WR-02: What's-New popup has no EN-fallback direction handling under RTL

**Files modified:** `assets/whats-new.js`, `assets/app.css`, `tests/42-whats-new-gating.test.js`
**Commit:** 092fca7
**Applied fix:** The help BLOCKER-2 analog, mirroring `help.js _fallbackIds` /
`help.css .is-en-fallback`:
- `entries()` now flags EN-fallback entries `_enFallback` on a SHALLOW COPY
  (never mutating the shared `CHANGELOG_CONTENT_EN` objects). Coverage goes
  beyond the review's per-entry sketch to also handle the whole-locale-load

  entry is a fallback AND `document.documentElement.dir === 'rtl'`.
- `app.css` gains `.whats-new-popup.is-en-fallback { direction: ltr;
  text-align: start; }` beside the popup block — scoped to the fallback class
  only, so native HE popups keep reading RTL.
- New T-42-V6 behavior gate in `tests/42-whats-new-gating.test.js`: RTL doc +
  missing HE entry → class present; RTL doc + native HE entry → class absent.
  Verified RED against the pre-fix `whats-new.js` (git-stash probe) before
  committing.

Zero-innerHTML discipline unchanged — all content still renders via
`createElement` + `textContent`.

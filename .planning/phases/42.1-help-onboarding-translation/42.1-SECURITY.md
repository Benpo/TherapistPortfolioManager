# Phase 42.1 — Security

> Per-phase security contract: threat register, accepted risks, and audit trail.

**Phase profile:** Localization / content-integrity phase (translate v1.3 help, changelog, welcome/tour copy into HE/DE/CS). No auth, network, storage, or crypto surface added. Every threat is a *content-integrity / injection* concern on strings that flow into the client DOM, plus offline (precache) availability and copy-quality governance. All 10 PLANs carried a `<threat_model>` block → register authored at plan time; ASVS L1 grep-depth verification is sufficient.

---

## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| translated `.js` content file → client DOM | Locale content files are executable JS shipped to clients; strings flow through help.js / changelog.js / whats-new.js render paths | Untrusted-if-corrupted UI copy |
| translator/agent → shipped copy | Copy corruption (clinical or business-internal framing, emoji, control chars) can leak into client-shipped strings | User-facing text |
| translated string → tour / popup DOM | `\n\n` and `{ui:}`/`{n}`/`{total}`/`{screen}` placeholder tokens are structural; dropping them corrupts layout/labels | Structural tokens |
| new content file → service-worker precache | A file missing from `PRECACHE_URLS` breaks help/changelog offline | Offline availability |
| page `<script>` tags → popup locale data | Popup reads `window.CHANGELOG_CONTENT_<loc>`; a page missing the locale sibling tag renders English regardless of the `entries()` fix | Locale selection |
| localized version label → hardcoded EN fallback | Dropping `{V}`/`{X.Y}` silently routes to an English label with no gate | Version string |
| EN-fallback marker → RTL css | The css keys LTR-only off the `.is-en-fallback` marker; a wrong marker flips a native section LTR or an EN-fallback section RTL | Text direction |
| hardcoded chrome string → user | Any chrome / aria / tour screen-name string left hardcoded ships English to a Hebrew user | UI copy |
| first non-EN render → real browser | jsdom has no layout; RTL/bidi + `pre-line` rendering only observable in a real engine | Render correctness |
| human sign-off → ship | Sapir/Ben's Hebrew read is the human backstop for copy quality before Phase 43 | Copy quality |

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pre-prod pipeline → prod deploy branch / docs-gate anchor | pre-prod must never write `deploy` or move the `Deploy from <sha>` anchor |
| public `*.pages.dev` origin → the internet | pre-prod is openly reachable; exposure is controlled by noindex + unguessable name + license gate |
| repo working tree → pre-prod staged tree | anything copied in is shipped to a public host |

| T-44-08 | Tampering / Availability | pre-prod writing the deploy branch or docs-gate anchor | high | mitigate | Trigger isolation (pre-prod branch only, never main) + target isolation (force-push deploy-preprod only) + no docs-gate step (D-08); reviewer confirms no `deploy`/`main` reference (Pitfall 3) |
| T-44-09 | Information Disclosure | public pre-prod origin | medium | mitigate | `X-Robots-Tag: noindex` + unguessable `*.pages.dev` name + license gate + no real data (D-09); no Cloudflare Access (a login wall would degrade iOS standalone-PWA cold-launch/SW testing) |
| T-44-05 | Information Disclosure | staged tree leak into pre-prod | high | mitigate | Shared whitelist transform + a retained Verify-no-sensitive-files guard on the pre-prod staged output |

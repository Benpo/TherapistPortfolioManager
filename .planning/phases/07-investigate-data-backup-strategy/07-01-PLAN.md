1. Download JSZip 3.10.1 minified to `assets/jszip.min.js`:
   ```bash
   curl -o assets/jszip.min.js https://cdnjs.cloudflare.com/ajax/libs/jszip/3.10.1/jszip.min.js
   ```

2. Create `assets/backup.js` as a self-contained module exposing `window.BackupManager` with these functions:

**`exportBackup()`** — Async. Collects all clients + sessions from PortfolioDB. For each client with a `photo` data URL:
  - Parse MIME type from data URL: `const match = photo.match(/^data:(image\/\w+);base64,(.+)$/)`
  - Determine extension from MIME: `image/png` -> `.png`, `image/jpeg` -> `.jpg`, default `.jpg`
  - Add to ZIP as `photos/client-{id}{ext}` using `{ base64: true, compression: "STORE" }` (images already compressed)
  - Replace `client.photo` with the filename reference `photos/client-{id}{ext}`
  - If no match (not a data URL), leave photo as-is (could be a filename reference already)

  Build manifest object:
  ```javascript
  { version: 1, exportedAt: new Date().toISOString(), appVersion: "1.0",
    clients: clientsClean, sessions: allSessions,
    settings: { language: localStorage.getItem("portfolioLanguage"),
                theme: localStorage.getItem("portfolioTheme") } }
  ```

  Add `backup.json` to ZIP with `{ compression: "DEFLATE" }`.
  Generate ZIP blob. Return `{ blob, filename }` where filename is `sessions-garden-backup-YYYY-MM-DD.zip`.

**`triggerDownload(blob, filename)`** — Creates object URL, triggers download via `<a>` click, revokes URL. Sets `localStorage.portfolioLastExport = Date.now()`.

**`importBackup(file)`** — Async. Accepts a File object. Detects format by extension:
  - If `.json`: read as text, parse JSON, call `normalizeManifest()`, then restore.
  - If `.zip`: read as ArrayBuffer, `JSZip.loadAsync()`, extract `backup.json`, parse, normalize manifest. For each client whose `photo` starts with `photos/`: find the file in ZIP, read as base64, reconstruct data URL using the file extension to determine MIME (`jpg` -> `image/jpeg`, `png` -> `image/png`). If photo file missing in ZIP, set `client.photo = null` gracefully.

  Restore strategy: **Replace** (clear all, then add). Before clearing, this function does NOT show UI — the caller (overview.js in Plan 02) handles the confirmation dialog.

  After restoring clients and sessions, restore settings from manifest if present:
  - `settings.language` -> `localStorage.portfolioLanguage`
  - `settings.theme` -> `localStorage.portfolioTheme`
  Skip `portfolioBackupSnoozedUntil` and `portfolioLastExport` (session-specific).

  After restore, set `localStorage.portfolioLastExport = Date.now()` (importing IS a fresh data state).

**`normalizeManifest(manifest)`** — Handles version compatibility:
  - No `version` field (old JSON format): wrap as `{ version: 0, clients: manifest.clients || [], sessions: manifest.sessions || [] }`. Photos are already inline base64 — no transformation needed.
  - `version: 1`: current ZIP format, return as-is.

**`sendToMyself(blob, filename)`** — Async. Calls `triggerDownload(blob, filename)` first. Then opens mailto with subject "Sessions Garden backup - {date}" and body instructing user to attach the downloaded ZIP. Use `window.location.href = mailto:...`.

**`pickBackupFolder()`** — Async. Guards with `if (!("showDirectoryPicker" in window)) return null`. Calls `window.showDirectoryPicker({ mode: "readwrite" })`. On AbortError (user cancelled), return null. On success, store handle in module-scoped variable `_savedDirHandle`, set `localStorage.portfolioAutoBackupEnabled = "true"`, return handle.

**`autoSaveToFolder(blob, filename)`** — Async. If `_savedDirHandle` is null, return false. Query permission, request if needed. Create file handle, write blob, close writable. Return true on success, false on error (log warning).

**`isAutoBackupSupported()`** — Returns `"showDirectoryPicker" in window`.

**`isAutoBackupActive()`** — Returns `_savedDirHandle !== null`.

Use IIFE pattern matching existing codebase style (like app.js). No ES modules, no imports.

  <action>
Create 3 new files:

**assets/i18n-disclaimer.js** — Self-contained translation object (window.DISCLAIMER_I18N) with all 4 languages (en, he, de, cs). Must include:
- Page title ("Terms of Use")
- Section headings and content for each legal section:
  1. Not Medical — this is an organizational/administrative tool for energy balancing practitioners, not a medical device, not a substitute for medical advice
  2. Data Privacy — all data stays local on user's device only, developer has zero access, no data collection whatsoever
  3. Backup Responsibility — user is solely responsible for backing up their data via the app's export function
  4. No Liability — tool provided as-is, no warranty on data integrity or availability
  5. Single-User License — license is personal and non-transferable
  6. Widerrufsrecht / Withdrawal Right — per §356 Abs. 5 BGB
- General terms checkbox label (e.g. "I have read and accept the Terms of Use")
- Widerrufsrecht checkbox label — CRITICAL: must use the exact LG Karlsruhe-compliant text from RESEARCH.md:
  - DE: "Ich stimme der Vertragsausfuehrung vor Ablauf der Widerrufsfrist zu und weiss, dass dadurch mein Widerrufsrecht erlischt."
  - EN: "I agree to the execution of the contract before the expiry of the withdrawal period and acknowledge that my right of withdrawal expires upon commencement of the download/access."
  - HE: the Hebrew version from RESEARCH.md
  - CS: the Czech version from RESEARCH.md
- Accept button text
- Receipt download button text
- Language selector labels
- Receipt content strings (title, accepted date, language, legal text summary)

**disclaimer.html** — Full-screen gate page. Structure:
- Uses same CSS (tokens.css + app.css) for visual consistency with the app
- Has its own theme detection script (same inline pattern as app pages) at top of <head>
- Language selector at top-right (dropdown with 4 languages), auto-detects browser language via navigator.language on load
- Centered content card with "Sessions Garden" branding at top
- Sectioned legal content — each section has a heading and paragraph, displayed in the selected language
- Two checkboxes at the bottom of the content:
  1. General terms acceptance checkbox (unchecked by default)
  2. Widerrufsrecht/withdrawal checkbox (unchecked by default, separate and distinct)
- "Accept" button — disabled until BOTH checkboxes are checked
- After acceptance: show a "Download Receipt" button and a "Continue to App" button
- RTL support: set dir="rtl" on html when Hebrew is selected, dir="ltr" otherwise
- The page must NOT load db.js, app.js, or any app page scripts — only tokens.css, app.css, i18n-disclaimer.js, and disclaimer.js
- Add a "Terms of Use" link in footer text for re-access (disclaimer.html?readonly=true shows content without checkboxes/buttons when terms already accepted)

**assets/disclaimer.js** — Page logic:
- On load: check if ?readonly=true — if so, render content only (no checkboxes, no buttons)
- On load: detect language from navigator.language (map "he"/"de"/"cs" prefixes, default to "en"), check localStorage portfolioLang as fallback
- Language selector onChange: re-render all text, update dir attribute for RTL
- Checkbox change handlers: enable Accept button only when both are checked
- Accept handler:
  1. Store portfolioTermsAccepted = new Date().toISOString() in localStorage
  2. Store portfolioTermsLang = selected language in localStorage
  3. Hide checkboxes/accept button, show receipt download + continue buttons
- Receipt download: generate plain text .txt file via Blob + createObjectURL pattern from RESEARCH.md. Include: app name, acceptance timestamp, language, license key (if available from localStorage, else omit), summary of accepted terms. Filename: sessions-garden-acceptance-YYYY-MM-DD.txt
- Continue handler: redirect to ?next parameter URL, or ./index.html if no ?next

  <files>index.html, sessions.html, add-client.html, add-session.html, reporting.html</files>
  <action>
Modify all 5 existing HTML pages to add the terms gate check as the VERY FIRST script in the head, BEFORE the existing theme detection script.

Add this inline synchronous script to each page:
```html
<script>
  (function(){try{if(!localStorage.getItem('portfolioTermsAccepted')){var n=encodeURIComponent(window.location.pathname+window.location.search);window.location.replace('./disclaimer.html?next='+n);}}catch(e){})();
</script>
```

This must be:
- The absolute first `<script>` tag in `<head>` (before the theme detection script)
- Synchronous (no defer, no async)
- Inline (not external file)
- Uses window.location.replace() (not .href) so back button doesn't loop

The existing theme detection script stays as the second script. No other changes to these pages in this task.

Verify the gate works by: if portfolioTermsAccepted is not set in localStorage, opening any of the 5 pages should immediately redirect to disclaimer.html with the correct ?next parameter.

<objective>
Close the three geometry UAT gaps on the shipped tour engine: the first-paint spotlight offset (gap 2), the step-to-step jumpiness (gap 3), and the RTL BLOCKER where an off-center anchor's spotlight/tooltip mirrored to the wrong physical side in Hebrew (gap 4). Root cause (diagnosed in 41-UAT.md): positionSpotlight writes PHYSICAL getBoundingClientRect coordinates (r.left = viewport-left) into LOGICAL inset properties (insetInlineStart), which in RTL resolves to the right edge — so any anchor that is not full-width mirrors; and the spotlight's `transition: all` animates width/height/inset from the fresh element's 0×0 base on every mount, so the ring visibly grows into place. Step 1's full-width greeting card hid the RTL bug; step 2's off-center button exposed it.

Purpose: this is the most fragile surface in the milestone (D-14 flags the WebKit bug class that bit Phases 37 and 40). Fixing coordinates + first-paint here, before the storyline recomposition (41-10) reorders the steps, keeps the geometry correct for whatever anchors the new spine lands on.
Output: physical-coordinate positioning + first-paint snap in tour.js/tour.css, and a strengthened WebKit probe that can no longer pass while the bug is present.
</objective>

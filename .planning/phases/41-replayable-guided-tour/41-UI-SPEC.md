# Phase 41 — UI Design Contract

> Visual and interaction contract for the **Replayable Guided Tour**. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope:** Phase 41 delivers the real tour engine that Phase 26 designed as a mockup. This contract
> is the visual + interaction source of truth for the ~10-step tour, its tethered-tooltip / bottom-sheet
> chrome, the D-11 fallback modal, the mid-tour exit choice, the coordinator-governed re-entry reminder,
> and the finish card. **Every value is inherited from the existing garden design system** — this phase
> introduces **no new palette, no new type scale, no new spacing rhythm**. It constrains how the tour
> surfaces compose existing `tokens.css` semantic tokens and the `.help-root`-style local scale the
> sketch already ported into `help.css`.
>
> **Reuse note for the checker:** this is a *reuse-only* phase against an approved parent contract
> (`26-UI-SPEC.md`) + a live sketch (`.planning/sketches/003-tour-fallback/`). Values below cite their
> upstream source. See "Checker Sign-Off — Accepted Exceptions" before flagging greenfield rules.

## Spacing Scale

Authored in `rem` (1rem = 16px), 4px base grid. Tour surfaces reuse the `.help-root` local scale verbatim.

| Token | rem | px | Usage in this phase |
|-------|-----|-----|---------------------|
| xs | 0.25rem | 4px | Icon-to-label gap, step-counter margin, arrow inset |
| sm | 0.5rem | 8px | Tooltip inner row gaps, button vertical padding, fallback-location spacing |
| md | 1rem | 16px | Tooltip padding (block + inline), button-row top margin, control gaps |
| lg | 1.5rem | 24px | Fallback-modal scrim padding, finish-card content block gap |
| xl | 2rem | 32px | Finish-card vertical rhythm |
| 2xl | 3rem | 48px | Finish-card hero-to-actions separation (large viewports) |
| 3xl | 4rem | 64px | Reserved — not required by tour surfaces |

Exceptions: **`--tap-target-min: 44px`** is an accessibility constraint (Apple-HIG minimum tap target), **not** a spacing-rhythm step — it governs every interactive tour control (Next / Previous step / Close / "Take me there" / exit-choice / reminder / finish actions). Tooltip `min-width: 16rem` / `max-width: 22rem` are sizing constraints (see Interaction Contract), not rhythm tokens.

## Typography

Constrained 4-role scale, inherited from `26-UI-SPEC.md` and the `.help-root` tokens. No new sizes.

| Role | Size | Weight | Line Height | Used for |
|------|------|--------|-------------|----------|
| Body | 1rem (16px) | 400 | 1.6 | Tour step descriptions, fallback-modal body, finish-card body, reminder-card body |
| Label / UI | 0.9rem (≈14px) | 700 | 1.4 | Step counter, all tour buttons ("Next"/"Done"/"Previous step"/"Close tour"), "Take me there" link, exit-choice buttons, reminder Start/Dismiss |
| Heading | 1.25rem (20px) | 700 | 1.3 | Tour tooltip title, fallback-modal title, reminder-card title |
| Display | 1.75rem (28px) | 700 | 1.2 | Finish-card headline only |

**Rules:**
- Exactly these 4 roles. No new font sizes introduced.
- **Exactly two font weights: 400 (body) and 700 (labels, headings, display).** Do not use 500, 600, or 800 in tour surfaces. (The 600 face is present in the project but unused here — this satisfies the checker's 2-weight rule; see Accepted Exceptions.)
- Body line-height **1.6** deliberately (not 1.5) — the tour is calm long-form reading for a non-technical audience; matches the existing help long-copy precedent.
- Step **titles and headings authored as nouns / gerunds, never imperatives** (Pitfall 5; Phase 24 D-05) — e.g. "Adding a client", "The heart of a session" — so the Hebrew translation (D-11) produces natural noun/infinitive forms.

## Color

Inherited from `assets/tokens.css` semantic tokens (auto-resolve in `[data-theme="dark"]`). The 60/30/10 discipline maps onto the garden palette. **Use the listed token vars, never literal hex.** Sketch 003's color discipline is a hard rule: **only the forward action and the spotlight ring take the accent; every other control stays neutral.**

| Role | Token (use this var) | Light value | Usage in this phase |
|------|----------------------|-------------|---------------------|
| Dominant (60%) | `--color-modal-overlay-bg` dim + underlying page | `rgba(26,24,35,0.4)` | The inert dim overlay is the dominant field during the tour; the live page shows through the spotlight hole |
| Secondary (30%) | `--color-surface` / `--color-surface-alt` | `#ffffff` / `#fdf7ed` | Tooltip surface, fallback-modal card, bottom-sheet, finish card, reminder card |
| Accent (10%) | `--color-primary` | `#2d6a4f` garden green | **Reserved list below — never "all interactive elements"** |
| Neutral controls | `--color-surface-secondary-btn` / `--color-text` / `--color-text-muted` | `#e8f5ee` / `#2f2d38` / `#5f5c72` | Previous step, Close tour, "Take me there", exit-choice, reminder Dismiss |
| Destructive | `--color-danger` | `#ea4b4b` | **Not used** — the tour has no destructive actions (see Copywriting) |

**Accent (`--color-primary`) reserved EXCLUSIVELY for:**
1. The tour **spotlight ring** — `box-shadow: 0 0 0 4px var(--color-primary), 0 0 0 9999px var(--color-modal-overlay-bg)` around the anchored element.
2. The tour **Next / Done** button (the single forward action per step) — filled primary, `--shadow-button`, `--color-text-inverse` label.
3. The finish card's **one primary action** ("Add your first client" — the single lead forward action; "Start your first session" and "Browse the help center" stay neutral to preserve one-accent-per-surface).
4. **Focus-visible outline**: `2px solid var(--color-primary)`, `outline-offset: 2px` on every interactive tour element.
5. (Coordinator reminder) The reminder card's **Start** button is the one accent control on that surface; **Dismiss** is neutral.

Everything else — Previous step, Close tour, the fallback "Take me there" link, the two exit-choice buttons, the bottom-sheet's non-forward controls — uses **neutral/surface tokens**. The "I'll explore myself" exit path is visually first-class but **not** accent-colored (respects autonomy; non-forced tone).

**Dark mode:** every surface reads dark token vars via `[data-theme="dark"]` — no separate dark treatment authored. The dim overlay, spotlight ring, and all cards resolve automatically. No literal hex anywhere in the tour layer.

## Copywriting Contract

Tone (D-03, DNA-EN): **calm garden voice — warm, unhurried, clinician-domain fit. Explicitly NOT generic SaaS product-tour energy** ("Let's get you set up! 🚀"). All copy is `data-i18n`-keyed and ships in **all 4 locales (EN/HE/DE/CS) in this phase** (D-11). EN below is the canonical **draft**; final wording passes the TOUR-01 pipeline (writer grounded in real UI labels → factual gate → native-speaker gate → DNA/consistency gate, reusing the Phase 39 D-19/D-20 shape). Hebrew uses noun/infinitive forms (Phase 24 D-05).

**Step copy contract (D-01 / D-03):** title + **1–2 calm sentences, ≤ ~40 words** per step. Each step names a **real, always-present** UI label (chrome-only anchors, D-02). Draft EN for the 10-step full-spine route:

| # | Anchor (chrome) | Draft title | Draft body (≤~40 words) |
|---|-----------------|-------------|-------------------------|
| 1 | Overview / page heading | "Welcome to your garden" | "This is your quiet home base. Let's walk through running a session, end to end — at your own pace. You can leave any time." |
| 2 | "Add Client" button (`nav.addClient`) | "Adding a client" | "Every journey starts with a person. This is where you add a new client to your garden." |
| 3 | Start-a-session entry | "Starting a session" | "When you sit down to work, you'll open a session from here." |
| 4 | Session form — setup zone | "Setting the scene" | "You begin by choosing the client, the date, and how you met — in the clinic, online, or another way." |
| 5 | Session form — the heart (emotions / severity + Heart-Wall) | "The heart of a session" | "Here you capture the trapped emotions, their severity, and whether a Heart-Wall is present. This is the core of the work." |
| 6 | Session form — save + after-save | "Saving and keeping it safe" | "When you save, the session is stored on this device. You can also export any session as a PDF — look for this export icon." *(tooltip shows the colored export glyph inline; sessions list is empty on a fresh app, so export is named + shown, not demoed live)* |
| 7 | Sessions page nav | "Looking back" | "All your saved sessions gather here, ready to browse, filter, and revisit." |
| 8 | Reporting dashboard nav | "Seeing the whole picture" | "The dashboard gently surfaces patterns across your work over time." |
| 9 | Backup (cloud button) | "Keeping a copy" | "Your data lives only on this device — so a backup matters. This is where you export a safe copy." |
| 10 | "?" help entry + finish | "You're ready" | "That's the whole path. The help center is always here if you need it — you'll find it under this question mark." |

| Element | Copy (EN canonical, draft) | i18n key (existing / new) |
|---------|----------------------------|---------------------------|
| Step forward button | "Next" / final step "Done" | `help.tour.next` / `help.tour.done` |
| Step back / close | "Previous step" / "Close tour" | `help.tour.back` / `help.tour.close` |
| Step counter | "Step {n} of {total}" | `help.tour.counter` |
| **Fallback (anchor missing) — required, never silent (D-11 / TOUR-02)** | "This is on the {screen} screen." + link **"Take me there"** | `help.tour.fallbackBody` / `help.tour.takeMeThere` |
| **Mid-tour exit choice — concise (D-08)** | Prompt: "Leaving the tour?" · Option A: **"Remind me later"** · Option B: **"I'll explore myself"** | `help.tour.exit.title` / `help.tour.exit.remindLater` / `help.tour.exit.exploreMyself` (new) |
| **Re-entry reminder card (next session, coordinator-governed, D-08)** | Title: "Ready to finish the tour?" · **"Start"** / **"Dismiss"** | `help.tour.reminder.title` / `help.tour.reminder.start` / `help.tour.reminder.dismiss` (new) |
| **Finish card (D-10)** | Headline: "You're all set" · Body: "Your garden is ready. Whenever you like, take the first small step." | `help.tour.finish.title` / `help.tour.finish.body` (new) |
| Finish action — primary (accent) | "Add your first client" | `help.tour.finish.addClient` (new) |
| Finish action — neutral | "Start your first session" | `help.tour.finish.startSession` (new) |
| Finish action — neutral link | "Browse the help center" | `help.tour.finish.helpCenter` (new) |
| "?" popover new entry (D-13) | "Take the tour" | `help.entry.takeTour` (new) |
| Welcome CTA (rewire target, D-12) | "Take the guided tour" (unchanged label) | `help.welcome.ctaTour` (exists) |

**Destructive actions in this phase: NONE.** The tour creates, deletes, and mutates no data. `--color-danger` is intentionally unused. Closing the tour is non-destructive (offers the exit choice, never a "delete" confirm).

**Microcopy rules:**
- No emojis in any tour copy (project-wide convention).
- No bare single-word controls except the established "Next" / "Done" forward pair — back/close are noun phrases ("Previous step", "Close tour"), pairing symmetrically with "Take me there".
- Exit-choice phrasing must stay **concise** — button-length, not paragraphs (D-08).
- The fallback modal **names where the thing lives** ("This is on the Clients screen.") — a soft landing, never an error tone.
- No imperative-only EN that forces unnatural Hebrew imperatives (Pitfall 5; Phase 24 D-05).

## Interaction Contract (phase-specific — consumed by checker, planner, executor)

Hard constraints the executor must honor. Geometry values are ported from the working sketch 003.

**Accessibility — minimum tap target:** every interactive tour control has a **44×44px minimum hit area** via `--tap-target-min` (`min-height`/`min-width` on `.tt-btn`, `.tt-link`, exit-choice, reminder, finish actions). Focus-visible = `2px solid var(--color-primary)` + `2px` offset on all of them.

| Surface | Contract |
|---------|----------|
| **Tour step schema (D-01, D-02, TOUR-02)** | Each step: `{ id, page, anchor: '[data-tour="…"]', i18nKey, takeMeThereHref, screenName }`. Anchor binds ONLY to dedicated `data-tour="…"` attributes (never `#id`, `:nth-child`, or text — locale-independent, RTL-stable, refactor-safe). Anchor selection mechanism + the anchor-presence rot-guard test shape are planner's discretion (TOUR-02). |
| **Degradation resolution (D-02, TOUR-02 — the rot guard)** | anchor present & `offsetParent !== null` → **spotlight + tethered tooltip**; anchor missing / hidden → **centered fallback modal** with the SAME step text + a working **"Take me there"** link (NEVER a silent skip). A brand-new empty app must show spotlights on all 10 chrome anchors — the fallback is a rot guard, not the routine first-run experience. An anchor-presence test guards against rot. |
| **Spotlight** | `position: fixed`, `border-radius: var(--radius-md)`, `pointer-events: none`, `box-shadow: 0 0 0 4px var(--color-primary), 0 0 0 9999px var(--color-modal-overlay-bg)`, `transition: all 0.32s cubic-bezier(.4,0,.2,1)`. Ring = accent; the `9999px` shadow is the inert dim. |
| **Tethered tooltip (D-04)** | `position: fixed`, `z-index: 9100`, `min-width: 16rem`, `max-width: 22rem`, flexible height, `--color-surface` bg, `1px --color-border`, `--radius-md`, `--shadow-modal`, **logical-property padding only** (`padding-block`/`padding-inline: var(--space-md)`). Fade+slide in (`opacity`/`transform`), animated re-position via `inset-block-start`/`inset-inline-start` transitions. |
| **Arrow (D-04, RTL flip)** | 14×14 rotated square, `--color-surface` with `border-inline-start`+`border-block-start` = `--color-border`, positioned via `data-arrow="top|bottom|none"` + logical `inset-inline-start: var(--arrow-x)`. **RTL arrow-flip is a validated sketch behavior — keep it** (logical props flip for free). |
| **Small-screen bottom-sheet (D-05)** | Below a width breakpoint, the tooltip becomes a bottom-anchored sheet (`inset-inline: 0`, bottom-pinned, `border-start-*-radius` only, no arrow) — spotlight still highlights the anchor; no tooltip-collision math on cramped viewports. Breakpoint value is planner's discretion (recommend ~600–640px). |
| **Page inert during tour (D-07)** | The dim overlay blocks ALL clicks outside the tour chrome (Previous / Next / Close, and "Take me there" in fallback). Tour state can never desync from user actions underneath. Reuse the `app.js` modal scroll-lock idiom (~1591). How the inert overlay interacts with page scroll when steps 4–6 scroll the form into view is planner's discretion. |
| **Auto-navigate on Next (D-06, TOUR-03)** | When the next step lives on another page, Next navigates there itself (`location.href`); tour state `{tourId, stepIndex}` persists in **sessionStorage** and resumes on the next page's init. Key names follow `sg.*` conventions (planner's discretion). Pages the route touches: `index.html`, `add-session.html`, `sessions.html`, `reporting.html` — the engine loads on all of them (**sw.js PRECACHE_URLS + manual CACHE_NAME chore follow-up applies** to the new `assets/*.js`). |
| **Relaunch semantics (D-09)** | Relaunch **always restarts from step 1**. sessionStorage resume serves ONLY cross-page continuation inside a single run — never across launches. No offer-to-resume. |
| **Language re-render (D-11, TOUR-04)** | Subscribe to the `app:language` CustomEvent (app.js:126). On mid-tour switch, **cleanup-then-replace** (sketch Variant C pattern) — re-render tooltip/fallback text + chrome strings, re-flip tether/arrow for RTL, re-measure geometry. Must survive EN↔HE(RTL)↔DE↔CS in place with no overflow (`max-width` + flexible height). |
| **Mid-tour exit choice (D-08)** | Closing mid-tour offers a concise two-option choice: **"Remind me later"** → registers a low-precedence card with the Phase 40 `AttentionCoordinator` for next browser session; **"I'll explore myself"** → sets a persistent "never remind" flag (`sg.*`, planner's discretion). The tour itself still only ever RUNS on explicit click (TOUR-01 intact) — the reminder card offers to start, it does not auto-run. |
| **Re-entry reminder surface (D-08)** | One `AttentionCoordinator.register({id, eligible(), show()})` call at **low precedence** (below the security note — exact slot in `PRECEDENCE` is planner's discretion, documented). One-per-session marker, demo-off gate inherited. Must not tangle with the pending security-note cadence-backoff todo. |
| **Launch surfaces (D-12, D-13, D-16)** | (a) Welcome overlay CTA **rewired** from `./help.html` to launching the real tour (attention-coordinator.js:180, `help.welcome.ctaTour`). (b) New **"Take the tour"** row appended to the "?" popover extension slot (`app.js initHelpEntry` ~466–565; suggested position: after "Onboarding screen"/"Replay welcome"). (c) The "?" tour entry **hides in demo** (`window.name === 'demo-mode'`) — consistent with the Phase 35/40 demo seam. |
| **Finish card (D-10)** | Garden-voice card with a **"Browse the help center"** link PLUS action buttons **"Add your first client"** / **"Start your first session"**. One accent control only (Add-your-first-client, per the color discipline); the other two neutral. |
| **RTL / dark** | Logical properties only (`*-inline-*`, `inset-inline-*`); `[data-theme="dark"]` reads dark token vars; no physical `right:`/`left:`, no literal hex. Verified per D-14: Chromium-first dev loop, **Playwright-WebKit gate** for the TOUR-04 RTL criterion + spotlight/tooltip geometry, plus one real-Safari UAT pass. Modern evergreen baseline, no legacy shims (D-15). |

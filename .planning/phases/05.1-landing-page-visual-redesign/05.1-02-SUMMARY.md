## Deviations from Plan

**1. [Rule 1 - Bug] Fixed i18n renderer: textContent -> innerHTML**
- **Found during:** Task 1 (wiring icons into i18n system)
- **Issue:** The existing renderer used `icon.textContent = f.icon` (line 442 of original landing.js). Setting textContent with an SVG string would display the raw SVG markup as text, not render it as an icon.
- **Fix:** Changed to `icon.innerHTML = f.icon` so the SVG string is parsed and inserted as DOM nodes.
- **Files modified:** assets/landing.js
- **Verification:** Confirmed renderer now sets innerHTML; SVG will parse correctly in all 4 language switches
- **Committed in:** 33f6fe1 (Task 1 commit)

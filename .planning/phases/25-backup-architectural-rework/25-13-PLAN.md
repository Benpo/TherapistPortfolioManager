**Important constraint for UAT-E1 (cloud color clash):** The cloud icon's `--fresh` state currently uses `background-color: var(--color-success-bg)` which is the same hue family as the settings-gear active-state. On `settings.html`, both the gear (active-page indicator) AND the cloud (fresh-backup indicator) read as primary signals — the user can't tell which is "you are here." The UI-SPEC sets the gear as the load-bearing current-page signal, so the cloud's `--fresh` state must visually subordinate to it. Choose ONE of these patterns and apply consistently:
- Pattern A — non-fill ring: cloud `--fresh` switches from filled background to a thin ring/border (`background: transparent; border: 2px solid var(--color-success-border)`), keeping fill-treatment for `--warning` and `--danger` where stronger signal is warranted.
- Pattern B — different hue: cloud `--fresh` uses a desaturated/teal tint distinct from the gear's saturated green.

Pattern A is the more defensible choice (preserves color semantics: stronger fill = more urgent state). Choose Pattern A unless a strong CSS-token constraint forces Pattern B.

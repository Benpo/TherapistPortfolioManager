<objective>
Close 6 Settings-page UX gaps from `25-UAT-FINDINGS.md`:

- **UAT-A** — Custom-days input should appear ONLY when frequency='custom'; today it remains visible for daily/weekly/monthly.
- **UAT-B** — Password-acknowledgement checkbox sits to the left of a 2-line verification text; visual hierarchy reads wrong. Move checkbox underneath the text.
- **UAT-C2** — Optimize-all confirm dialog renders literal `{n}` and `{size}` placeholders. The dialog uses `App.confirmDialog({messageKey})` which sets `data-i18n` on the message element but never substitutes the placeholders. Fix the substitution path.
- **UAT-D1** — Backup-folder picker should be REMOVED from Settings → Backups entirely (Ben's intent: Phase 25 uses browser download / Web Share / mailto; folder picker is dead UI). NOT relocated, removed.
- **UAT-D3** — Schedule frequency change has insufficient feedback. Add a save-toast or visible success-pill.
- **UAT-D4** — Optimize-all completion toast is too small/far/brief. Either anchor it next to the Optimize button or persist the success message inline ≥5s.

All 6 are pure UX/code fixes on the Settings page; no cross-cutting architectural change. UAT-C2 specifically is a code bug in how `App.confirmDialog` consumes its `messageKey` — it does not run placeholder substitution. Fix once, benefit everywhere.

Out of scope: Hebrew copy for the new save-toast key — that lives in Plan 25-11 unless it lands first. (Plan 25-12 depends on 25-11 to avoid i18n key collisions; if 11 hasn't landed, this plan must add the keys itself in all 4 locales.)

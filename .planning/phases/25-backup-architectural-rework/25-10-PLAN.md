Output:
1. A separate `ALLOWED_SENTINEL_KEYS` whitelist that includes `snippetsDeletedSeeds`.
2. A dedicated raw-write path on `PortfolioDB` (e.g. `_writeTherapistSentinel`) that does NOT pass through `setTherapistSetting` (which coerces `customLabel`/`enabled` fields).
3. Reordered import flow: sentinel write executes BEFORE the snippet-restore loop so the subsequent `seedSnippetsIfNeeded` call sees the restored deleted-ids.
4. A behavior test that round-trips a non-empty sentinel through exportâ†’importâ†’post-restore-seed-check and verifies the deleted snippets stay deleted.

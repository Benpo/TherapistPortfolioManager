Refactor target — current exportEncryptedBackup signature (backup.js:643-671):
  Returns true (encrypted+downloaded), false (user pressed Skip → caller does unencrypted), 'cancel' (user cancelled).
  Problem: when result is `true`, the encrypted blob is no longer accessible to the caller — it was downloaded inside the function and the local `encBlob` went out of scope. The Share button cannot Share what it cannot see.

NEW return shape (object — additive; existing callers can be migrated):
  Resolves to: { ok: boolean, skip: boolean, cancelled: boolean, blob: Blob | null, filename: string | null }
  Cases:
    - User confirms encrypt + download succeeds: { ok: true, skip: false, cancelled: false, blob: encBlob, filename: encFilename }
    - User pressed Skip Encryption:                { ok: false, skip: true, cancelled: false, blob: null, filename: null }
    - User pressed Cancel/X/Esc:                   { ok: false, skip: false, cancelled: true, blob: null, filename: null }

  Backward compatibility: the existing overview.js openExportFlow (Plan 02) checks `encrypted === 'cancel'` and `encrypted === false` — Plan 08 updates THAT call site to the new object shape. No other call sites consume exportEncryptedBackup (verified by grep before refactoring).

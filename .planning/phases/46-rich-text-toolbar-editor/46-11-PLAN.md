<objective>
Two focused toolbar fixes from the 46-08 gate. Both touch rich-toolbar.js, so they ship together.

- **Gap 4 — Preview toggle label + icon (both surfaces).** The icon-only eye button is insufficient. Ben's spec: the button shows its TARGET state — eye icon + "Preview" label while editing (click goes to preview), pencil icon + "Edit" label while previewing (click goes back to editing). This applies wherever the toggle appears: the 7 focus-attached note fields AND the persistent export-editor bar (both are the same RichToolbar preview control, so one change covers both).

- **Gap 8 — Snippet-accept Enter must not fire list auto-continue.** Inside a list, accepting a snippet autocomplete with Enter currently BOTH expands the snippet (snippets.js handleKeyDown) AND continues the list (rich-toolbar.js onFieldKeyDown) — because both listeners handle the same keydown and snippets does not stop propagation. Fix direction (Ben-ratified): the list auto-continue YIELDS when the snippets autocomplete is active — snippet accept wins; plain Enter (no suggestion open) still continues the list. This is the RTXT-09 danger zone, so the snippets change is strictly additive.

Purpose: make preview state legible and stop the snippet/list Enter collision.
Output: a labelled target-state preview toggle on both surfaces, and a list-continuation guard that yields to the snippets popover, with regression tests for both Enter paths.
</objective>

<objective>
Gap 7 — emotions before/after must become an opt-out (pre-selected), not forced. Ben (verbatim): "in the export, we always export the emotions before/after, but I want to replace this with pre-selected box for this and not do it forced (several customers complained)." This is a pre-existing behaviour (not a Phase 46 regression) riding this gap round because the phase already touches the export/PDF pipeline. Source: `.planning/todos/pending/2026-07-14-export-emotions-optout-checkbox.md`.

Root cause found in source: the export Step 1 already has a "Choose which sections to include" mechanism (`#exportStep1Rows`), and the emotions before/after block IS the `issues` (issues + before/after severity) section — but its Step-1 row defaults UNCHECKED and, more importantly, does NOT gate the PDF: `buildRenderInputs()` always forwards `getIssuesPayload()` to `drawSeverityBlock`, and the clipboard copy (`buildSessionMarkdown`) always emits the Issues section. That "always" is Ben's forced-inclusion complaint.

**PLAN-APPROVAL NOTE (locked proposals Ben ratifies at the Task-1 checkpoint):**
- **Placement:** reuse the EXISTING Step-1 "Choose which sections to include" row for the emotions/issues section — no new standalone checkbox, no settings toggle. Flip its default to CHECKED (pre-selected opt-out) and make it actually gate output.
- **Scope:** the row controls the emotions before/after block = the PDF two-bar severity block (fed by the forwarded `issues[]`) AND the clipboard-copy Issues section. Both gate on the same choice.
- **Persistence:** remember the choice per therapist in localStorage (default = included). Sticky so repeat-complaint customers stay opted out, and so the copy path (which has no Step 1) can honour it too. Alternative offered at the checkpoint: per-export reset (simpler, but the copy path then always includes).
- **Wording:** clarify the row so it reads as "emotions before/after ratings" rather than only "Issues addressed" — exact label ratified at the checkpoint.

Purpose: stop forcing content customers don't want, without changing the default behaviour.
Output: a pre-selected, persisted opt-out that gates both export paths.
</objective>

<objective>
Build `assets/text-edit.js` (`window.TextEdit`) — the single undo-safe insertion chokepoint plus the pure string transforms that power the entire toolbar. This is the phase keystone (RESEARCH Pattern 1): `document.execCommand('insertText')` is the ONLY API that mutates a textarea while preserving the browser's native undo stack AND firing a real `input` event, and every later plan (toggle D-04, auto-format RTXT-03, renumber D-11, undo/redo D-20) routes through it.

Purpose: isolate the genuinely-tricky logic (undo/caret/selection math) into one tested module so the toolbar and mount plans are thin wiring. All transforms are side-effect-free `(value, selStart, selEnd) → {value, selStart, selEnd, replacement}` functions (RESEARCH §Validation Architecture), unit-testable in the zero-npm node runner; the thin `editInsert` shim applies the result in-browser.

Output: `assets/text-edit.js` + two node test files covering RTXT-01/03/05 logic.
</objective>

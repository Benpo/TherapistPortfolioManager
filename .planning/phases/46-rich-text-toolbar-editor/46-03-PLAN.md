<objective>
Build `assets/rich-toolbar.js` (`window.RichToolbar`) — the focus-attached formatting toolbar chrome plus its inline-formatting actions. This plan delivers the toolbar DOM, focus tracking, dock positioning, button active-state, the inline actions (bold/italic toggle, bullet/numbered insert, heading dropdown), desktop keyboard shortcuts (Ctrl/Cmd+B/I), and undo/redo buttons — all routed through the `window.TextEdit` chokepoint from 46-01. List auto-format mechanics and the live preview pane land in 46-04 (same module, next wave). Mounting onto the real note fields / export editor happens in 46-05 / 46-06.

The module is a shared, mountable toolbar following the vanilla IIFE `window.*` convention (like snippets.js). It takes a set of textareas and a config (headings on/off) so both add-session (7 note fields) and export-modal Step-2 (full toolbar incl. headings) reuse one instance.

Purpose: deliver RTXT-01 (markers via toolbar) + RTXT-02 (desktop shortcuts) + D-04/D-20 as thin wiring over the tested TextEdit transforms, honoring the UI-SPEC visual contract and the RTL physical-coordinate rule.
</objective>

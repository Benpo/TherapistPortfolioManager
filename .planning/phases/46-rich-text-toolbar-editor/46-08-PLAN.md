  <name>Task 1: Real-browser / real-device / real-PDF phase verification</name>
  <action>Pause for a human checkpoint: drive the built surfaces on the real browser/device matrix (installed Safari desktop + iPhone PWA, Chrome, Firefox) and a real opened PDF, working through the checklist below. Do not close the phase until every item passes or failures are routed to gap-closure.</action>
  <what-built>The full rich-text toolbar editor: focus-attached toolbar over the 7 note fields, desktop shortcuts, auto-format + nested lists + auto-renumber, per-field live preview, undo/redo buttons, and a redesigned export Step-2 editor (full toolbar, info note, 50%/maximize, mobile full-screen). Italic-in-PDF shipped true or flattened per 46-02.</what-built>
  <how-to-verify>
    Run the app on the pre-prod environment (Phase 44) on a real installed Safari PWA (desktop) AND a real iPhone PWA, plus desktop Chrome and Firefox. Verify:

    1. Toolbar + markers (RTXT-01, D-04): in a note field, select text and click Bold then Italic — markers wrap; click again on the wrapped selection — they unwrap; with no selection, click Bold — a marker pair appears with the caret between them (never a doubled-marker artifact). The toolbar docks above the focused field and hides when focus leaves all note fields. On the iPhone the toolbar is the only formatting affordance and scrolls horizontally on overflow (D-02).
    2. Shortcuts (RTXT-02, D-02): on desktop, Ctrl/Cmd+B and Ctrl/Cmd+I toggle bold/italic. On the iPhone there is NO keyboard shortcut path (buttons only).
    3. Auto-format + nested lists (RTXT-03, RTXT-05, D-09/D-10): type "- " then text + Enter — the list continues; type "1. " — an ordered list; Enter on an empty top-level item exits the list; Enter on an empty NESTED item outdents one level; Tab/Shift+Tab indent/outdent a list line; Tab on ordinary text moves focus (no keyboard trap).
    4. Auto-renumber acceptance (D-11, Ben's scenario): make a 4-item numbered list; delete item 3 — items renumber so the old 4 reads 3; cut/paste that item to the end — it takes number 4. The caret stays where expected and native undo (Ctrl+Z / the Undo button) reverts cleanly.
    5. Undo/redo buttons (D-20): the toolbar Undo/Redo buttons produce the SAME result as keyboard undo/redo on desktop; on the iPhone the buttons are the only undo affordance and work. FAILURE ROUTING (RESEARCH A4 / Pitfall 3): if button-undo does NOT match keyboard-undo on ANY target — most likely the iOS Safari PWA where `execCommand('undo')` can be unreliable — this is NOT an undifferentiated gap. It triggers a specific NAMED follow-up plan implementing the RESEARCH-prescribed fallback: a single module-level undo stack in window.TextEdit that ALSO intercepts Ctrl+Z (never a dual-stack), swapped in behind the existing button/keydown handlers with the pure transforms and the rest of the toolbar untouched. Record the exact failing target(s) in the SUMMARY so that follow-up is scoped precisely.
    6. Live preview (RTXT-04, D-05/06/07): toggle the eye — a preview pane appears below the field and re-renders as you type; empty field shows the empty-state copy; leaving the field resets the toggle.
    7. Snippets + autogrow UNCHANGED (RTXT-09 — the danger zone): type a snippet trigger in an enhanced note field — it still expands; the field still autogrows as content grows; a toolbar insert also autogrows and does not break snippet expansion.
    8. Export Step-2 (D-03/D-08/D-16/D-17/D-18): open the export modal, go to Step 2 — the full toolbar (with headings) is present; the ephemeral-edit info note is shown; the window opens at ~50% and the maximize toggle expands it to ~90%; the editor fills the space (no tiny fixed textarea); the Edit/Preview swap works; on the iPhone Step 2 is a full-screen keyboard-aware takeover; edits here do not save to the session.
    9. Real PDF (D-13/D-14): export a session with italic Hebrew AND Latin, bold, and a nested list; OPEN the real PDF. If true italic shipped (46-02 GREEN/AMBER), italic renders slanted for both scripts; if it flattened (D-14), italic is upright and the disclosure note/tooltip is present. Bold + nested lists render; Hebrew RTL/bidi is correct.
    10. RTL: switch to Hebrew — the toolbar, heading dropdown, and preview mirror correctly (physical-coordinate positioning; no off-center anchor).
  </how-to-verify>

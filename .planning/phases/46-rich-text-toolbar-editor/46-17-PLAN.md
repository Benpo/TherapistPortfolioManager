<objective>
Round-3 gap fix for Phase 46. Both gaps were reproduced live in Playwright WebKit on build e0c48c5 and diagnosed with confirmed root causes in 46-UAT.md (gaps 12 and 13). This plan fixes BOTH with scoped JS changes to `assets/rich-toolbar.js` only, proven RED→GREEN by a new jsdom unit test (Gap 12) and a new WebKit probe assertion set (Gap 13 + Gap 12 end-to-end).

**Gap 12 — persistent-bar controls are DEAD until the editor is focused once (high).** `_dispatch`/`togglePreview` resolve their target field from `_focused` (the last-focused registered field) and return early when it is null. That "visible bar ⇒ focused field" invariant held for the focus-attached shared bar but was broken by the round-1 `persistent: true` mount: the export bar is ALWAYS visible, so users click its controls BEFORE ever focusing the editor (default page focus sits on `#sessionDate`). Every control on the persistent bar (bold, lists, undo, preview…) silently no-ops on first click; the mousedown+preventDefault focus-preservation also means the click itself never focuses the editor. Fix: dispatch on a persistent bar resolves the target from the clicked control's OWN bar (the `_persistentBars` map already knows field↔bar) and focuses that field as part of the action — so the persistent bar never depends on prior focus. Note-field shared-bar behaviour unchanged.

**Gap 13 — export preview pane opens 100% below the scroll fold (high).** With the editor focused, Preview DOES open, but the pane is inserted `afterend` the editor inside the fixed-height `.export-edit-area` (overflow-y:auto), and the editor (flex-fill, min-block-size 240px) already fills/overflows the visible area — measured pane top 659.6px vs area visible bottom 652.0px at 1440x820: the pane starts entirely below the fold. The user sees no change. On note fields the page grows so the pane is naturally visible — the export's scroll container is the only surface with a fold. Fix: when the preview opens for a field whose scroll container is the export edit area, bring the pane into view — scroll the container so the pane top is visible below the sticky bar. Must not disturb note-field preview, the sticky pinned bar, or undo/caret state; RTL-safe.

Purpose: make every persistent-export-toolbar control work on first click and make Preview actually reveal its pane, without changing any of the seven note-field behaviours.
Output: two scoped JS edits to `assets/rich-toolbar.js`, a new RED-first jsdom test, and a new RED-first WebKit probe assertion set.

**Explicitly NOT in this round:**

- No i18n keys, no sw.js change, no new shipped asset, APP_VERSION stays 1.4.0.

<objective>
Close WR-06 (GATE-04): the release-moment rule can silently and permanently self-disable.

Today `scripts/docs-gate.js` computes `versionChanged = oldVer && newVer && oldVer !== newVer`, where `oldVer`/`newVer` come from a LOCAL `extractAppVersion()` regex `/APP_VERSION\s*[:=]\s*['"](\d+\.\d+\.\d+)['"]/`. If that regex ever stops matching `assets/version.js` (APP_VERSION renamed, re-quoted with backticks, moved, or reformatted), both endpoints return null, `versionChanged` is false, and GATE-04's entire release-moment check is skipped — with no warning, no tripwire, permanently. Nothing asserts the extractor still parses the live file. This is precisely the "docs rot with no tripwire" failure class the phase exists to prevent, sitting inside the phase's own release-detection logic.

The fix (per the review's scoped remedy): (1) lift `extractAppVersion` into a shared lib `scripts/lib/version-parse.js` so the gate and the invariant use ONE implementation (D-17); (2) add a FIFTH invariant `checkVersionParse(repoRoot)` to `scripts/lib/invariants.js` that reads the live `assets/version.js` from disk and throws if `extractAppVersion` returns null; (3) have the gate's Phase-1 `runInvariants()` call it, so a format drift fails the NEXT push closed instead of silently disabling the release rule. Because this changes gate behavior, a falsifiable behavior test (D-21) ships in the same plan.

This plan runs in wave 2 and depends on 43-09 only because both edit `scripts/docs-gate.js` (the wave rule forbids same-file plans in the same wave); the edits are in different regions (43-09: satisfier/trailer readers; this plan: the extractor import and the runInvariants call).

Purpose: give GATE-04 a tripwire so it cannot self-disable on a benign version.js refactor.
Output: a shared extractor, a fifth invariant, a gate wired to both, and a behavior test.

Out of scope (deferred, rationale): the GATE-04 live-ship proof at v1.3.0 milestone close remains a human/milestone-close item (43-07's SUMMARY records it open) — not closable at phase scope. WR-05 and IN-01..IN-05 are deferred — see 43-08's objective for the full list.
</objective>

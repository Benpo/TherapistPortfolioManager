<objective>
Close the phase's one Blocker (CR-01) and its paired doc gap (WR-02), both in the CI enforcement layer (GATE-02 "the unbypassable layer").

Today `.github/workflows/deploy.yml`'s docs-gate step decides "deploy branch absent" vs "present-but-unfetched" with `git ls-remote --exit-code --heads origin deploy`, but its `if … then … else …` conflates ALL non-zero exit codes into the bootstrap `else`. `ls-remote --exit-code` returns 2 when the branch is genuinely absent but 128 on a network/auth failure — so a transient CI network blip on an EXISTING deploy branch silently collapses the range to the tip commit only, gating just one commit of a multi-commit push. It is not self-healing: the next successful run anchors past the skipped commits, permanently un-gating them. This directly contradicts the phase goal ("the unbypassable layer") and the step's own header comment.

The fix: branch three ways on the captured exit code — rc=0 anchored, rc=2 bootstrap, any other rc → echo + exit 1 (fail closed) — per D-04. Because this is a change to gate BEHAVIOR, the project's standing rule (runtime-behavior code requires a falsifiable behavior test — D-21, feedback-behavior-verification) applies. Inline YAML shell cannot be unit-tested, so the range-resolution shell is lifted into one shared script (`scripts/ci-resolve-docs-range.sh`) that both `deploy.yml` and the test invoke — the same "one implementation, two callers" shape the gate itself already uses (D-01/D-17). deploy.yml becomes a thin caller.

WR-02 rides along in the same shell step: when the anchor cannot resolve (rewritten main history, mangled deploy subject), a `Docs-Emergency-Skip:` trailer cannot help because the shell runs before the gate does. The resolver prints a recovery runbook on those fail-closed branches and CLAUDE.md documents it.

Purpose: make GATE-02's "unbypassable / fail-closed" property true under a realistic infra fault, and remove the emergency-skip dead-end.
Output: a tested shared CI range resolver, a rewired deploy step, and a recovery note.

Out of scope for this gap-closure run (explicitly deferred, with rationale):

- IN-01..IN-05 (three-dot parseRange, dead `remote` var, unknown-arg fallback, duplicated normalize, dead covers[] entry) — Info tier; both wired callers use the correct forms today. Left for opportunistic cleanup.

branch commit message (`git log origin/deploy -1 --format=%s` → "Deploy from <SHA>"). Widen deploy.yml's

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local hook → CI | The local hook is bypassable (--no-verify); the CI step is the authoritative, unbypassable layer. |
| deploy branch anchor → CI range | The anchor is read from a commit message; a malformed/absent anchor must not silently widen or drop the range. |
| npm install → git config | prepare runs a git-config command on every install; must not fail the install in a non-repo context. |

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|

| T-43-17 | Denial of function | cancel-in-progress drops a push range → an unchangelogged change ships | medium | mitigate | OD-2 anchored range is self-healing: the next successful run re-covers any skipped commits. |
| T-43-18 | Tampering | anchor unresolvable → CI range wrong | medium | mitigate | Fail closed if the deploy branch exists but the anchor won't resolve; defined tip-only fallback ONLY for the benign no-deploy-branch bootstrap. |

<objective>
Close three verified Warning-tier fail-open / false-diagnosis paths in the gate's core matching logic, all in `scripts/docs-gate.js` (plus the single-source-of-truth predicate home, `scripts/lib/role-table.js`).

WR-01 (GATE-01/GATE-03): `scripts/docs-gate.js:185-186` decides "the help was edited" / "the changelog was edited" with `/(^|\/)help-content-(en|he|de|cs)\.js$/` and its changelog twin — UNANCHORED to `assets/`, while `role-table.js`'s `SATISFIER_RE` is anchored `^assets\/…`. Verified: the gate regex matches `tests/fixtures/help-content-en.js`, which role-table.classify returns `ignored` for. Editing such a path would silently satisfy the help/changelog demand for every trigger in the range — a fail-open divergence between the two modules D-17 exists to keep singular. Fix: role-table.js owns the anchored definition; the gate consumes it.

WR-03 (GATE-03): CLAUDE.md states the three trailer keys use "exact casing (case-sensitive)", but git's `%(trailers:key=…)` matcher is case-insensitive — verified: a lowercase `docs-emergency-skip:` bypasses the whole gate though CLAUDE.md promises rejection. The DoD contract's own rule is "the rules below are exactly what the gate enforces — nothing more, nothing less." We make the CODE match the written contract (post-filter returned trailers by exact key case) rather than weaken the doc — this preserves the deliberately title-cased key design and keeps the security-sensitive emergency skip exact-match. (The alternative — editing CLAUDE.md to say "case-insensitive" — was rejected: it would loosen the emergency-bypass surface and contradict the keys' intended design.)

WR-04 (D-14/OD-3): the two `%(trailers:…)` readers omit `unfold`, then split on `\n`. A folded multi-line trailer value — which git produces when an editor wraps a long line, and which CLAUDE.md explicitly encourages via multi-file trailers — is split into two values, mis-blocked as "missing a reason". Fix: add `unfold` to both format specifiers so a folded value returns as one line.

WR-03 and WR-04 touch the SAME two reader functions (`trailerValuesOverRange`, `trailerValueForCommit`), so they are fixed together: the readers switch from `valueonly` to emitting `Key: value` with `unfold`, then in JS keep only lines whose key matches the requested key with exact case, returning the value.

Purpose: make satisfier detection singular (WR-01), make the written trailer-case contract true (WR-03), and stop false-blocking correctly-authored folded trailers (WR-04). All three fixes preserve fail-closed direction.
Output: anchored role-table predicates, a corrected gate, and three new falsifying behavior cases.

Out of scope (deferred, rationale): WR-05, IN-01..IN-05, and the GATE-04 live-ship proof — see 43-08's objective for the full deferral list. WR-06 (release-moment invariant) is closed separately in 43-10.
</objective>

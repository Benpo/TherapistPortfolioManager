The four CSS moves (UI-SPEC "Net CSS deltas" + element-by-element map):
1. Remove the two tinted-card backgrounds/radii/card-pad from
   .backup-modal-section--contents and .backup-test-password-card so every
   section is a plain block on the modal surface (Fork 1 = Variant A, plain +
   dividers). KEEP the test-card's display:flex; flex-direction:column;
   gap:var(--space-md,16px) — that is the section's internal stacking rhythm,
   not a card device, and a test asserts on it.
2. Collapse heading sizes to one: .backup-modal-section-heading 1.4rem ->
   1.125rem, and DELETE the .backup-modal-section--import
   .backup-modal-section-heading font-size:1.125rem override (redundant).
3. Recolour .backup-modal-import-warning from danger tokens -> warning tokens,
   and align its box metrics with .backup-test-password-result so the two
   inline bands read as one component (one severity palette).
4. (Consistency) the two inline bands share an identical box treatment so they
   cannot drift again.

The one TOKEN move (checker Warning 2 — user-confirmed "Fix it now", 2026-06-14):
5. Add dark-mode --color-warning-bg and --color-warning-text overrides inside
   the existing [data-theme="dark"] block in assets/tokens.css. Today that
   block overrides only the success palette (--color-success-bg/text/border,
   tokens.css ~181-183); --color-warning-* and --color-danger-* have NO dark
   override, so the amber band currently renders its LIGHT values (#fff3cd bg /
   #856404 text) against the #202828 dark surface — low contrast. Moving the
   Import warning into the amber family (move 3) makes this gap matter for the
   Import band too, not just the pre-existing test-password error band. Adding
   the two dark overrides fixes BOTH bands in one place (they share the tokens).

2026-05-16) by removing round-by-round visual accretion, AND make the resulting
amber band honest in dark mode. The design thinking is already done and
user-confirmed in 27-UI-SPEC.md — this plan executes it plus the one dark-mode
contrast correction the plan-checker surfaced and the user approved.

dark-block overrides only). No other file changes. The pre-commit hook will

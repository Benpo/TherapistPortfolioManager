    **Step B — assets/snippets-editor.js (or extend assets/settings.js)** — implement these flows:

    1. **Init flow:**
       - On DOMContentLoaded (after App.initCommon resolves):
         - Read `Snippets.getPrefix()` → prefill `#snippetPrefixInput`.
         - Wire prefix input `change` handler → validate locally → call `Snippets.setPrefix(value)` → catch validation errors and render localized message into `#snippetPrefixError`.
         - Call `renderSnippetList()`.
         - Subscribe to `document.addEventListener("app:snippets-changed", renderSnippetList)`.

    2. **renderSnippetList():**
       - Read `App.getSnippets()` for the current cache.
       - Apply search filter: substring-case-insensitive match on `snippet.trigger` OR on `snippet.expansions[currentLang]`.
       - Apply tag filter: keep snippets whose `tags` intersects the selected tag-chip set (empty set = no filter).
       - Sort: alphabetical by trigger.
       - Render each row into `#snippetList`:
         ```
         <div class="snippets-list-row" data-snippet-id="...">
           <span class="snippets-list-trigger">prefix+trigger</span>
           <span class="snippets-list-preview">first 60 chars of currentLang expansion (textContent)</span>
           <div class="snippets-list-tags">[tag chips]</div>
           <button class="icon-button" data-action="edit" data-snippet-id="..." aria-label="Edit">✎</button>
           <button class="icon-button danger" data-action="delete" data-snippet-id="..." aria-label="Delete">🗑</button>
         </div>
         ```
       - Render the tag-filter chip row: deduplicate all tags across snippets, render as toggleable chips above the list. Active chips → `.is-active` class.
       - Toggle `#snippetListEmpty` visibility based on filtered count.

    3. **Add / Edit modal flow (D-12):**
       - Click `#addSnippetBtn` → openEditor({}); click row edit → openEditor(snippet).
       - openEditor(snippet?) populates:
         - `#snippetEditorTrigger` ← snippet.trigger || ""
         - `#snippetEditorActiveLangLabel` text ← `App.t("snippets.editor.expansion.lang." + currentLang)` (or fallback "Expansion (" + currentLang.toUpperCase() + ")")
         - `#snippetEditorActiveExpansion` ← snippet.expansions[currentLang] || ""
         - Tags chips ← snippet.tags || []
         - "Edit translations" block starts hidden; clicking toggles it open and renders 3 other-locale textareas, each with a filled/empty dot:
           ```
           <div class="snippets-editor-translation">
             <label>
               <span class="snippets-editor-locale-dot" data-state="filled|empty"></span>
               <span>{locale name}</span>
             </label>
             <textarea id="snippetEditorExp_{locale}">...</textarea>
           </div>
           ```
         - Show "Delete" button only when editing existing snippet (snippet.id present).
         - Show "Reset to default" button ONLY when snippet.origin === "seed" AND has been modified (updatedAt > createdAt OR contents differ from SNIPPETS_SEED entry).
       - Save handler:
         - Read trigger, tags, all 4 locale textareas.
         - Build candidate snippet: `{ id: snippet.id || generateId(), trigger, expansions: { he, en, cs, de }, tags, origin: snippet.origin || "user" }`.
         - Validate trigger uniqueness across cache (case-insensitive, exclude self if editing).
         - Call `PortfolioDB.validateSnippetShape(candidate)` — surface localized error message in `#snippetEditorTriggerError` if it throws.
         - On success: `PortfolioDB.addSnippet` or `updateSnippet`; await; `App.refreshSnippetCache()`; broadcast `{ type: 'snippets' }`; close modal; show toast `snippets.toast.saved`.
       - Delete handler: `App.confirmDialog` (reuse Phase 22 modal) → if confirmed, `PortfolioDB.deleteSnippet(id)` → refresh + broadcast + close + toast.
       - Reset-to-default handler: `App.confirmDialog` → `PortfolioDB.resetSeedSnippet(id)` → refresh + broadcast + close + toast.
       - Cancel handler: close modal, discard unsaved edits silently if clean; if dirty (compare to opened snippshot), reuse Phase 22 dirty-confirm pattern (`confirm.discard.*` keys from Plan 03).

    4. **Tag chip input (D-17):**
       - Type in `#snippetEditorTagsTextInput` → on Enter / comma / Tab keypress, push current value (lowercased, trimmed) as a chip into `#snippetEditorTagsList`. Skip if duplicate (case-insensitive).
       - Each chip is `<li class="snippets-tag-chip">{tag}<button data-action="remove-tag" aria-label="Remove">×</button></li>`.
       - As user types, show `#snippetEditorTagsSuggestions` populated with existing tags (across all snippets) whose lowercase starts with the input value; click suggestion → add as chip.
       - Suggestions: max 8, sorted alphabetically, exclude already-chipped tags.

    5. **Import / Export (REQ-8):**
       - Export click → build `payload = { version: 1, exportedAt: ISO, snippets: App.getSnippets().filter(s => s.origin === "user" || (s.origin === "seed" && s.updatedAt > s.createdAt)) }` → download via blob:
         ```
         const blob = new Blob([JSON.stringify(payload, null, 2)], {type: "application/json"});
         const url = URL.createObjectURL(blob);
         const a = document.createElement("a"); a.href = url;
         a.download = "snippets-" + new Date().toISOString().slice(0,10) + ".json";
         document.body.appendChild(a); a.click(); a.remove();
         URL.revokeObjectURL(url);
         ```
       - Import click → trigger hidden `#importSnippetsFile` click; on file selected:
         - FileReader → JSON.parse → wrap in try/catch.
         - Validate shape: must have array `snippets`; each entry passes `PortfolioDB.validateSnippetShape`.
         - Check for duplicate triggers WITHIN the imported file → reject with localized error.
         - Check for collisions with existing snippets (same `trigger`, case-insensitive) → if zero collisions, proceed to apply immediately. If ≥1 collision, open `#snippetImportModal` with per-trigger Replace/Skip toggles.
       - Collision modal rendering:
         ```
         <div class="snippets-import-collision-row" data-trigger="...">
           <span class="snippets-import-collision-trigger">{prefix}{trigger}</span>
           <div class="snippets-import-collision-toggle" role="group">
             <button type="button" data-choice="replace" class="is-active">Replace</button>
             <button type="button" data-choice="skip">Skip</button>
           </div>
         </div>
         ```
         Default selection: Replace. Toggle on click.
       - Apply button:
         - For each imported snippet:
           - If no collision: `addSnippet`.
           - If collision + choice=Replace: `updateSnippet` (preserving existing id but overwriting fields).
           - If collision + choice=Skip: no-op.
         - After all rows: refresh cache, broadcast, close modals, toast `snippets.toast.imported` with count.

    6. **Reset all modified seeds:**
       - Button → confirm modal → for each snippet where origin='seed' AND (updatedAt > createdAt OR differs from SNIPPETS_SEED entry), call `PortfolioDB.resetSeedSnippet(id)`.
       - Refresh + broadcast + toast count.

    7. **Search input debounce:** `input` event with 150ms setTimeout → renderSnippetList.

    8. **Prefix validation message** — localized: read `Snippets.setPrefix(value)` throw error code, map to i18n key like `snippets.error.prefix.tooShort`, `snippets.error.prefix.tooLong`, `snippets.error.prefix.invalidChar`. Set text via `App.t(key)` into `#snippetPrefixError`.

    9. **Demo-mode parity:** the Settings UI works identically in demo mode (no demo-specific guards). Verify on demo_portfolio DB.

    10. **Performance:** for users with thousands of snippets, the list rendering is full-redraw per filter change. v1 sticks with flat rendering — virtual scrolling deferred per CONTEXT/SPEC out-of-scope. If perf becomes an issue, refactor in v1.2.

# Plan 24-04 — Browser UAT (caret popover + textarea wiring)

**Why this exists:** the caret-mirror popover positioning cannot be unit-tested cleanly — the math depends on real `getComputedStyle`, real `getBoundingClientRect`, real textarea scroll behavior, and on RTL flips. Plan 04 Task 3 ships with all 11 automated regex tests green, but the positioning math gets verified here.

**Pre-conditions:**
- Local dev server running (`python3 -m http.server 8080` or equivalent).
- Browser console open (so you can see errors).
- Hard-reload (`Cmd+Shift+R` / `Ctrl+Shift+R`) to pick up sw.js v121+ precache.
- Test data: at least one client in the DB so you can reach `add-session.html`.

**Task 3 status rule:** every scenario below must produce the expected outcome. If even one fails, the task stays `in_progress` — do NOT mark Plan 04 complete. Re-run scenarios after fixing each defect.

---

## Note before you start

Plan 04 ships the engine, the seed pack, and the CSS. The actual textarea wiring (`data-snippets="true"` attribute) + the `<script src="./assets/snippets.js">` tags on every page that needs them are **added in Task 4**. UAT verification of the user-visible behavior should be performed AFTER Task 4 is committed.

If you're running this UAT before Task 4 ships, scenarios 1–12 will all fail with "no popover appears" — that's expected, not a bug. Re-run after Task 4.

## Scenario 7 — Keyboard nav: ArrowDown / ArrowUp

1. In any wired textarea, type `;` then `b` (so multiple candidates appear, e.g., `;betrayal`, `;bitterness`, `;blaming`, `;b…`).
2. Press **ArrowDown** twice.
3. Press **ArrowUp** once.

**Expected:** The highlighted row moves down by 2 on the two ArrowDown presses, then up by 1 on ArrowUp — selection wraps from last back to first if you go past the end. The selection ring is visually obvious.

**Pass / Fail:** ___

# Phase 24 Plan 08 — Browser UAT script

Tests the nav-guard wiring on top-of-page logo (brand-link) and bottom "Back to
Overview" button. Both targets must trigger the unsaved-changes confirm dialog
when the form is dirty, and navigate silently when clean.

## Scenario 1 — New session, brand-link logo (clean → dirty → cancel → confirm)

1. Open `add-session.html` (no `?sessionId=` — new session flow).
2. **Clean state**: do not type anything. Click the top-left **Sessions Garden**
   logo. **Expected:** navigates to `./index.html` with no dialog.
3. Return to `add-session.html`. Pick a client, type something in any field
   (notes, comments, etc.). The form is now dirty.
4. Click the logo. **Expected:** "Leave this session?" confirm dialog appears
   with "Leave without saving" / "Stay on this session" buttons.
5. Click "Stay on this session". **Expected:** stays on the form, no navigation.
6. Click the logo again. Click "Leave without saving". **Expected:** navigates
   to `./index.html`.

**Pass:** clean → silent navigate. Dirty → dialog. Cancel keeps you on the
page. Confirm navigates and bypasses the beforeunload handler (no double-dialog).

## Scenario 2 — New session, Back to Overview button (same matrix as Scenario 1)

Repeat Scenario 1 but click the bottom **Back to Overview** button instead of
the logo. Expected behavior is identical — guard fires on dirty, silent on clean.

## Scenario 3 — Edit existing session, brand-link logo

1. Open `add-session.html?sessionId=<N>` for an existing session. Page loads
   in read mode.
2. Click pencil icon to enter edit mode.
3. Change something (e.g., flip session type from clinic to remote — Fix A from
   commit 30c4fd9 ensures this triggers dirty state).
4. Click the logo. **Expected:** confirm dialog.

**Pass:** dirty detection works on edit-existing flow too (already worked before
Plan 08 for the Cancel button; this verifies the new logo guard wires into the
same dirty predicate).

## Scenario 4 — Other pages (predicate is undefined)

1. Open `index.html` (overview). Click the logo. **Expected:** silent navigation.
2. Open `sessions.html`. Click the logo. **Expected:** silent navigation.
3. Open `settings.html`. Click the logo. **Expected:** silent navigation.

**Pass:** the global brand-link guard is a safe no-op on pages without a
dirty-able form. `window.PortfolioFormDirty` is `undefined` there, the isDirty
callback returns false, and the guard short-circuits to default click behavior.

Fix: when a block starts with a heading line, split the heading line from the rest and render them as two separate blocks.

       - Match `block` against `/^(#{1,3})\s+([^\n]*)(?:\n([\s\S]*))?$/`. Captures: $1 = hashes, $2 = heading text (first line only), $3 = optional remainder after the first newline.

       ```
       // D-23 (Phase 24): single-newline paragraph behavior is LOCKED — consecutive non-blank
       //   lines within a paragraph render with <br> joins (line 52). Blank line = new paragraph.
       //   Matches standard CommonMark / GitHub flavor / Notion contract. Do NOT change.
       // D-24 (Phase 24): heading regex now accepts an optional body remainder after the
       //   heading line, so "## heading\nbody" renders as <h2>heading</h2><p>body</p> instead
       //   of <p>## heading<br>body</p>. Single-line headings unchanged.
       ```

    3. Do NOT change `applyInline`. Do NOT change `escapeHtml`. Do NOT change the blank-line block split logic (`escaped.split(/\n{2,}/)`). Do NOT alter the list rendering branch.

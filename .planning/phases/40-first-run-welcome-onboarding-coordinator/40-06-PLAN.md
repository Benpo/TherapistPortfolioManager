<objective>
Close the MAJOR UAT gap: the install nudge never appears on any Chromium session
because AttentionCoordinator.run() executes once at DOMContentLoaded BEFORE Chrome's
late-firing beforeinstallprompt, so installEligible() (= !!deferredPrompt || isMacSafari())
is always false at arbitration and the late event only stashes deferredPrompt without
ever re-arming the coordinator (confirmed Pitfall-1 race, 40-UAT.md test 3).

Fix direction (Ben-confirmed): in the beforeinstallprompt handler, after stashing
deferredPrompt, re-run arbitration. run() self-guards on the one-per-session marker
(D-02) and iterates PRECEDENCE in order (D-01), so the re-arm surfaces the nudge ONLY
when nothing else has claimed the slot this session and no higher-precedence surface is
eligible — preserving both invariants by construction.

Purpose: make the per-browser install affordance (ONBD-04) actually reachable while
keeping the single-surface-per-session guarantee (ONBD-03) intact.
Output: a re-armed handler + a falsifiable behavior test that fails without the re-arm.
</objective>

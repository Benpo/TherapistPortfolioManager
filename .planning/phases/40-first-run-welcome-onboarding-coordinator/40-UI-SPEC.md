# Phase 40 — UI Design Contract

> Visual and interaction contract for the First-Run Welcome & Attention-Surface Coordinator.
> Generated by gsd-ui-researcher.
>
> **Scope:** This phase ships **production code** implementing the previously-mocked welcome
> overlay plus new coordinator-governed surfaces. Every visual value below is *inherited from
> the existing garden design system* (`assets/tokens.css`, D-10 of Phase 26) — this contract
> introduces **no** new palette, scale, or typography. It constrains how the new/production
> surfaces compose existing tokens and reuses the locked welcome contract verbatim.
>
> **Authoritative upstream contract:** `.planning/milestones/v1.1-phases/26-in-app-onboarding-overview-help-system/26-UI-SPEC.md`
> governs the welcome overlay (spacing steps, 4-role type scale, color roles, CTA hierarchy,
> `help.welcome.*` copy, one-shot `sg.welcomeSeen` semantics). Phase 40 promotes that mockup to
> production using **Variant B (Split)** from `.planning/sketches/001-welcome-overlay/` (D-10)
> and adds three new surfaces (install nudge, mobile expectation hint, "Replay welcome" row).

## Design System

| Property | Value |
|----------|-------|
| Tool | **none** — vanilla HTML/CSS/JS PWA, zero npm deps, no build step (PROJECT.md hard constraint). shadcn gate **not applicable**: `components.json` absent, not React/Next/Vite, and a complete semantic-token system already exists in `assets/tokens.css`. |
| Preset | not applicable |

| Font | **Rubik** (self-hosted woff2, `font-display: swap`). New surfaces use **only weights 400 and 700** (per 26-UI-SPEC Typography — the 600 face used by existing chrome such as `.security-guidance-note` is NOT introduced into any Phase 40 surface). Fallback `system-ui, sans-serif`. |
| Coordinator UI | The attention-surface coordinator (D-06/D-07) is **logic, not a rendered surface** — it has no visual footprint of its own. It only decides *which* registered surface renders per session. It is invisible in demo mode (`window.name === 'demo-mode'`, D-09). |

## Spacing Scale

Inherited verbatim from 26-UI-SPEC (4px base grid, authored in `rem`, 1rem = 16px). New Phase 40
surfaces MUST use these steps; existing app components keep their current values (do not refactor).

| Token | rem | px | Usage in this phase |
|-------|-----|-----|---------------------|
| xs | 0.25rem | 4px | Icon-to-label gap, inline chip padding, popover row inset |
| sm | 0.5rem | 8px | Compact stacking (nudge body→actions gap, hint icon→text gap) |
| md | 1rem | 16px | Default element spacing; install-nudge card padding; welcome content block gap |
| lg | 1.5rem | 24px | Welcome overlay content block gap; card outer margin; mobile-hint padding block |
| xl | 2rem | 32px | Welcome overlay vertical rhythm (copy-side gap); split art-side padding |
| 2xl | 3rem | 48px | Welcome overlay art-side / copy-side padding (Variant B) |
| 3xl | 4rem | 64px | Welcome overlay copy-side vertical padding on large viewports |

Exceptions: none. The **44×44px minimum interactive size** is an accessibility constraint
(see Interaction Contract), not a spacing-rhythm token.

## Typography

Inherited verbatim from 26-UI-SPEC — exactly 4 roles, exactly 2 weights (400 / 700). No new sizes.

| Role | Size | Weight | Line Height | Used for |
|------|------|--------|-------------|----------|
| Body | 1rem (16px) | 400 | 1.6 | Welcome subtitle, install-nudge body, mobile-hint body |
| Label / UI | 0.9rem (≈14px) | 700 | 1.4 | CTAs, nudge/hint buttons, "Replay welcome" menu row, Safari install pointer link |
| Heading | 1.25rem (20px) | 700 | 1.3 | Install-nudge card title |
| Display | 1.75rem (28px) | 700 | 1.2 | Welcome overlay greeting headline only |

**Rules:**
- Exactly these 4 roles; no new font sizes.
- **Exactly two weights: 400 (body) and 700 (labels, headings, display).** Do not use 500, 600, or 800 in new surfaces. (Existing `.security-guidance-note` uses 600 — it is **not restyled** in this phase, D-05, so no conflict arises.)
- Body line-height **1.6** deliberately (matches existing long-copy precedent).
- All headings and control labels authored as **nouns / gerunds / noun-phrases, not bare imperatives** (Phase 24 D-05) so the Phase 42.1 HE translation produces natural noun/infinitive Hebrew: "Add Sessions Garden to your computer", "Replay welcome", "Got it" — never "Install!" / "Back".

## Color

Inherited from `assets/tokens.css` semantic tokens (auto-resolve in `[data-theme="dark"]`). This
table maps the 60/30/10 discipline onto the garden palette — implementation uses the token vars,
never literal hex.

| Role | Token (use this var) | Light value | Usage in this phase |
|------|----------------------|-------------|---------------------|
| Dominant (60%) | `--color-background` (+ body radial gradient) | `#fdf8f0` warm cream | Welcome overlay backdrop |
| Secondary (30%) | `--color-surface` / `--color-surface-alt` | `#ffffff` / cream | Welcome content panel, install-nudge card, mobile-hint bar, "?" popover |
| Accent (10%) | `--color-primary` | `#2d6a4f` garden green | **Reserved list below — not "all interactive elements"** |
| Destructive | `--color-danger` | `#ea4b4b` | **Not used in this phase** — Phase 40 has no destructive actions |

**Accent (`--color-primary`) reserved EXCLUSIVELY for:**
1. Welcome overlay **primary CTA** ("Take the guided tour") — filled primary button (`.cta--primary`).
2. Install-nudge **primary action** — the single filled **[Install app]** button (Chromium only). One accent element per card, max.
3. The active **"?" affordance state** (`.is-active`, mirrors `.settings-gear-btn.is-active`) — unchanged from Phase 39.
4. **Focus-visible outline**: `2px solid var(--color-primary)`, `outline-offset: 2px` (existing pattern, `app.css`) — on every interactive element in new surfaces.

**Everything else uses neutral/surface tokens:**
- Welcome **secondary CTA** ("I'll explore myself") — `--color-surface-secondary-btn` (`--color-green-100`), first-class but **not** accent-colored (respects therapist autonomy — Calm/non-forced).
- Install-nudge **dismiss** ("No thanks") and Safari **Add-to-Dock pointer link** — neutral text / `--color-text-muted`, surface hover.
- **Mobile expectation hint** carries **no accent at all** — it is calm/informational, not a call to action (D-16 "calm app voice, not a warning"). Its topic link is a plain underlined text link (`--color-text` / `--color-primary` link color per existing link convention), its dismiss ("Got it") is neutral. It MUST NOT reuse the deleted iOS banner's full-bleed `--color-primary` fill.

**Botanical illustration usage:** reuse existing repo art only — `assets/illustrations/watering-can.png`,
`garden.png`/`garden-2.png`, `hero-left/right.png`. Variant B **art-side** = one calm botanical hero
that must fit the split panel and dark mode. Dark mode: follow the established `invert(1) + screen blend`
botanical pattern (Phase 11) — no separate dark illustration set. The sketch's inline-SVG motif is a
**placeholder**; the build swaps in a real botanical asset (Claude's discretion per D-52/CONTEXT, must fit Variant B + dark).

## Copywriting Contract

Tone (D-14): **warmth of calm/wellness apps + clinician-domain fit** — explicitly NOT generic SaaS
tour energy. Reference user = Sapir (warm, calm, non-technical, Hebrew-primary). All copy is
`data-i18n`-keyed, **EN canonical, and ships in all 4 locales (EN/HE/DE/CS) in this phase** (UI-chrome
rule, CONTEXT language scope). Native-speaker agent review before ship; Sapir reviews tone.

### Welcome overlay — inherited verbatim from 26-UI-SPEC (do NOT re-author)

| Element | Copy (EN canonical) | i18n key |
|---------|---------------------|----------|
| Welcome headline (Display) | "Welcome to your garden" | `help.welcome.title` |
| Welcome subtitle (Body) | "Sessions Garden keeps your client work private, on this device, and organized around how you actually run a session." | `help.welcome.subtitle` |
| **Primary CTA** (accent) | "Take the guided tour" | `help.welcome.ctaTour` |
| Secondary CTA (first-class, neutral) | "I'll explore myself" | `help.welcome.ctaExplore` |

### "Replay welcome" menu row (new — appended to the Phase 39 "?" popover, D-17)

| Element | Copy (EN canonical) | i18n key |
|---------|---------------------|----------|
| Menu row label | "Replay welcome" | `help.entry.replayWelcome` |

Position: after "Help center", before "Contact us" (CONTEXT suggested; Claude's discretion). Reuses
`.help-entry-item` styling verbatim — a plain `.help-entry-item` row, no accent, no icon required
(may carry an optional inline-SVG glyph consistent with the popover's other rows). Replaying is a
**direct open** outside the coordinator's arbitration — it never re-arms `sg.welcomeSeen` or the
session marker (D-17).

### Install nudge (new — computer-focused, D-12/D-14)

| Element | Copy (EN canonical, draft — Sapir reviews) | i18n key |
|---------|--------------------------------------------|----------|
| Card title (Heading) | "Add Sessions Garden to your computer" | `onboard.install.title` |
| Card body (Body) | "Install it for a calm full-screen window and one-click access — it still works fully offline, and your data stays on this device." | `onboard.install.body` |
| **Primary action — Chromium only** (accent, real `beforeinstallprompt`) | "Install app" | `onboard.install.ctaInstall` |
| macOS Safari pointer (Body, shown instead of the button when no `beforeinstallprompt`) | "On Safari, open the File menu and choose Add to Dock." | `onboard.install.safariHint` |
| Safari help link (Label, neutral) | "See install help" → Phase 39 install help topic | `onboard.install.safariLink` |
| Dismiss (Label, neutral) | "No thanks" | `onboard.install.dismiss` |
| Reassurance line under dismiss (Body, muted) | "You can install any time from Help." | `onboard.install.laterHint` |

Rationale for "No thanks" over "Not now": dismissal is **gone forever** (D-14, persistent flag) —
"Not now" would falsely imply a re-ask. "No thanks" + the muted reassurance line makes the one-ask
honest and calm; the Help install topic is the evergreen fallback.

### Mobile expectation hint (new — all-mobile, one-shot, D-16)

| Element | Copy (EN canonical, draft — Sapir reviews) | i18n key |
|---------|--------------------------------------------|----------|
| Hint body (Body) | "Sessions Garden is designed for computers. On a phone your data lives on each device separately — it won't move between your phone and your computer." | `onboard.mobileHint.body` |
| Topic link (Label, neutral underlined) | "How Sessions Garden works on mobile" → Phase 39 D-16 mobile-expectations help topic | `onboard.mobileHint.link` |
| Dismiss (Label, neutral) | "Got it" | `onboard.mobileHint.dismiss` |

Voice: calm expectation-setting, **not a warning** (D-16). No accent fill, no alarm iconography.
Dismissed forever (one-shot persistent flag).

**Microcopy rules (unchanged from 26-UI-SPEC):**
- **No emojis** in any new copy (project-wide convention — note the deleted iOS banner's 📤 emoji does not carry forward).
- No imperative-only EN that forces unnatural Hebrew imperatives (Phase 24 D-05).
- No bare single-word controls — labels are noun/noun-phrases ("Install app", "Replay welcome", "Got it", "No thanks"), never "Back"/"OK"/"Install".
- Install content branches by platform capability (`beforeinstallprompt` present → real button; macOS Safari → File → Add to Dock pointer); **never** a fake universal button, and **never eligible** when already installed (`display-mode: standalone`).

### Destructive actions

**NONE.** No surface in this phase creates, deletes, or mutates data. `--color-danger` is intentionally
unused. (Deleting the legacy iOS banner is a code removal, not a user-facing destructive action.)

## Interaction Contract (phase-specific — consumed by checker, planner, executor)

Hard constraints the build phase must honor. Every new surface: **logical CSS properties only**
(`*-inline-*`, `inset-inline-*`), `[data-theme="dark"]`-aware token vars, no physical `left/right`,
no literal hex, `data-i18n`-keyed copy in all 4 locales.

**Accessibility — minimum interactive size:** every interactive element in new surfaces (welcome CTAs,
"Replay welcome" row, install [Install app] / dismiss, Safari link, mobile-hint link / dismiss) has a
**44×44px minimum tap target**, inherited from the existing global rule and matching the header controls.
Focus-visible = `2px solid var(--color-primary)` + `2px` offset on all of them.

| Surface | Contract |
|---------|----------|
| **Attention-surface coordinator** (D-01..D-09) | Logic only, no visual footprint. Walks the precedence list **as data** — Welcome > What's-New (registers in Phase 42) > security note > install nudge > mobile expectation hint — once per browser session, from `initCommon`, and shows the **first eligible** surface. Session already claimed → sessionStorage marker suppresses any second surface (page navigation never re-fires). Fully disabled in demo mode (`window.name === 'demo-mode'`). Backup-reminder banner + footer integrity nudge are **NOT governed** (independent). |
| **Winning-but-unrenderable rule** (D-08 — resolved) | The Overview-only security note's `eligible()` MUST return **false** when its container (`#security-guidance-container`) is absent on the current page. Effect: on a non-Overview deep-link landing it is simply not eligible, and the coordinator cleanly advances to the next eligible surface — an unrenderable winner **never consumes the session slot**. (Documented choice per D-08 discretion.) |
| **Welcome overlay** (D-10, implements 26-UI-SPEC) | **Variant B (Split)**: botanical **art-side** on one inline side (`--color-surface-alt`, `border-inline-end`), **copy-side** (headline + subtitle + two stacked CTAs) on the other; sides flip cleanly under RTL via logical properties; panel `max-width ≈ 960px`, stacks to single column below ~720px (art-side on top, `border-block-end`). Full-screen backdrop with the garden radial gradient. `role="dialog"` `aria-modal="true"`, body scroll-lock (reuse `app.js` modal idiom), Esc dismisses, either CTA dismisses. One-shot via `localStorage 'sg.welcomeSeen'` (absent ⇒ first-run; covers upgraders once, no migration). **Interim tour CTA** ("Take the guided tour") opens `help.html` until Phase 41 rewires it (D-11 — one-line change later). |
| **Upgrader collision** (D-03) | Showing the welcome ALSO records the current `window.AppVersion.APP_VERSION` as the last-seen What's-New version (key form the Phase 42 What's-New will read — planner names it with Phase 42 in mind), so no v1.3 What's-New popup follows next launch. Uses `APP_VERSION`, never the SW/integrity-token layer. |
| **Install nudge** (D-12/D-13/D-14) | Dismissable **card**, coordinator-governed at the second-lowest tier (eligible from the 2nd session on; welcome naturally wins launch 1). Placement: bottom **inline-end** anchored card (`inset-block-end` + `inset-inline-end`, logical), `--color-surface` bg, `--shadow-card`, `--color-border` outline, `max-width ≈ 380px` — calmer than a full-bleed banner. **Chromium:** capture `beforeinstallprompt` early (often pre-DOMContentLoaded), and the filled **[Install app]** button triggers the native dialog. **macOS Safari:** no button — show the File → Add to Dock pointer line + "See install help" link. **Never** eligible when `display-mode: standalone`. Dismiss = **persistent flag, gone forever**. Desktop-only in practice (mobile gets the hint instead). |
| **Mobile expectation hint** (D-15/D-16 — iOS banner successor) | One-shot, **all mobile** (phone-class detection, not iOS-only UA sniffing — approach is Claude's discretion), coordinator-governed at the lowest tier. Placement: bottom-anchored full-width hint bar (the mobile-native position the deleted banner occupied) but **neutral surface styling** — `--color-surface` bg, `--color-text` copy, soft top border/shadow; **no `--color-primary` fill, no warning tone**. Body copy + topic link (→ P39 mobile-expectations help) + "Got it" dismiss. Dismissed = **persistent flag, gone forever**. |
| **iOS banner retirement** (D-15) | The hardcoded-English per-session inline banner at `index.html:367–374` is **DELETED** (promotes installing on a deliberately-unsupported platform). Its per-session sessionStorage idiom informs the coordinator's session marker; nothing of its visual styling (accent fill, emoji) carries forward. |
| **"Replay welcome" entry** (D-17) | Appended to the Phase 39 "?" popover via the documented `initHelpEntry` extension slot. Reuses `.help-entry-item` verbatim. Reopens the exact same welcome overlay as a **direct open** — outside per-session arbitration, never re-arms the one-shot flag or coordinator. Phase 41 later adds its own "Take the tour" row (no rename churn). |
| **RTL / dark** | Every new surface: logical properties only, `[data-theme="dark"]` reads dark token vars, no physical `right/left`/`margin-left`, no literal hex. Variant B split sides, install-card corner anchor, and mobile-hint bar all RTL-mirror via logical inset/border properties with zero JS. |

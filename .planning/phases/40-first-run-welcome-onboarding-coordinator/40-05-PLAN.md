<objective>
Load the coordinator on every app page (before app.js), delete the legacy hardcoded-English iOS install banner, and precache the new coordinator asset so the installed PWA runs it offline.

Purpose: makes the coordinator actually run on the app pages that boot via initCommon (ONBD-03), retires the iOS banner that promoted install on a deliberately-unsupported platform (D-15, part of ONBD-04's reconciliation), and keeps the offline PWA whole. This plan touches only HTML + sw.js, so it runs in parallel with Plans 03 and 04.

ACCEPTED EXCLUSION (architect-gate, source-verified): report.html does NOT call initCommon (assets/report.js boots its own init() on DOMContentLoaded, report.js:390), so the coordinator script included on report.html is inert there until a page calls run() — it is added for precache/uniformity only. A session starting on report.html defers governed surfaces to the next navigation (session marker unset, no stacking risk). Deferred by design; do not wire report.js to the coordinator.
Output: 8 edited pages, index.html banner removed, sw.js precache updated, and two static tests.
</objective>

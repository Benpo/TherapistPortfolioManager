Add edit-client modal logic to `assets/add-session.js` inside the existing DOMContentLoaded handler.

1. **DOM references** (near other element refs at top):
   ```
   const editClientBtn = document.getElementById("editClientBtn");
   const editClientModal = document.getElementById("editClientModal");
   const editClientClose = document.getElementById("editClientClose");
   const editClientSaveBtn = document.getElementById("editClientSaveBtn");
   const editClientCancelBtn = document.getElementById("editClientCancelBtn");
   const editClientPhotoInput = document.getElementById("editClientPhoto");
   const editClientPhotoPreview = document.getElementById("editClientPhotoPreview");
   const editClientReferralSelect = document.getElementById("editClientReferralSource");
   const editClientReferralOther = document.getElementById("editClientReferralOther");
   ```
   Also: `let editClientPhotoData = "";`

2. **Show/hide edit button** in `updateClientSpotlight()`:
   - After the existing spotlight visibility logic, add:
   - If a valid existing client is selected (clientId is a number, not empty, not "__new__"): show editClientBtn
   - Otherwise: hide editClientBtn
   - This ensures the button ONLY appears for existing clients, never for new inline creation or empty selection.

3. **openEditClientModal(clientId)** function:
   - Get client from cache: `const client = getSelectedClient(clientId, clientCache)`
   - If not found, try `await PortfolioDB.getClient(clientId)` as fallback
   - Populate all modal form fields from client object:
     - editClientFirstName, editClientLastName, editClientBirthDate, editClientEmail, editClientPhone, editClientNotes
     - Set type radio buttons (iterate editClientType radios, set checked and toggle-card active)
     - Set referralSource (same logic as add-client.js: check if value is in select options, else use "other" + other input)
     - Set photo: `editClientPhotoData = client.photoData || ""`; if exists, show preview with src
   - Store `editingClientId = clientId` for save handler
   - Show modal: `editClientModal.classList.remove("is-hidden")`
   - Apply translations: `App.applyTranslations(editClientModal)`

4. **Setup toggle group** for `editClientTypeGroup` — call `setupToggleGroup("editClientTypeGroup")` (the function already exists in add-session.js).

5. **Photo input handler** for editClientPhoto:
   - On change: read file as dataURL, set editClientPhotoData, show preview (same pattern as inline photo).

6. **Referral source handler** for editClientReferralSelect:
   - On change: show/hide editClientReferralOther based on "other" value (same pattern as add-client.js).

7. **Save handler** (editClientSaveBtn click):
   - Read all field values from the modal form
   - Validate firstName is non-empty (show toast if empty, return)
   - Build displayName from firstName + lastName
   - Calculate age from birthDate
   - Get the existing client object: `const existing = getSelectedClient(editingClientId, clientCache)` or fetch from DB
   - Call `PortfolioDB.updateClient({ ...existing, name: displayName, firstName, lastName, birthDate, age, email, phone, notes, type, referralSource, photoData: editClientPhotoData, updatedAt: new Date().toISOString() })`
   - Refresh clientCache: `clientCache = await loadClients(editingClientId)` — this reloads all clients, re-selects the current one
   - Call `updateClientSpotlight()` — this refreshes the spotlight card with updated name/photo
   - Show toast: `App.showToast("", "toast.clientSaved")`
   - Close modal

8. **Close handlers**:
   - editClientClose click, editClientCancelBtn click, modal overlay click: hide modal, reset form fields, clear editClientPhotoData
   - Do NOT touch session form data — it must remain untouched.

9. **Edit button click handler**:
   - `editClientBtn.addEventListener("click", () => { const cid = parseInt(clientSelect.value, 10); if (cid) openEditClientModal(cid); })`

10. **Read mode consideration**: If the session is in read mode (viewing existing session), the edit client button should still work — it edits the CLIENT, not the session. Do NOT disable it in setReadMode.

IMPORTANT: The session form fields (date, issues, comments, etc.) must be completely untouched by any edit-client operation. The modal is independent — it only reads/writes client data.

# Phase 47 — UI Design Contract

> Visual and interaction contract for the session-section reordering phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Reuse-only phase.** This phase extends existing surfaces (Settings page, session form, export modal) inside an established vanilla-JS/CSS zero-build design system (`assets/tokens.css`). It introduces NO new tokens, NO new font, NO new component library. All tokens below are DECLARED-FROM-SOURCE (already shipped), not new proposals. See `## Accepted Exceptions` for the greenfield-checker calibration note (repo memory: `feedback-ui-checker-greenfield-false-positives.md`).

**Companion mockup (D-17 binding gate):** `47-mockups.html` — small interactive mockup (EN/HE RTL + light/dark) for the REMAINING visual-detail decisions. Structure/placement/reorder model are already locked by sketch `010-section-groups-concept` + CONTEXT D-01..D-16; the mockup covers only reorder-row look, drag/arrow affordances, the ⓘ severity-off explainer, the skipped-rating (—) widget, and the topic + before-rating row. **✓ APPROVED 2026-07-23** (Ben, with Sapir on the Hebrew side) — D-17 gate closed; planning may proceed. ## Design System

| Property | Value |
|----------|-------|
| Tool | none (vanilla JS/CSS, zero-build, zero-npm — no shadcn; stack is not React/Vite/Next) |
| Preset | not applicable |
| Component library | none — hand-authored IIFE `window.*` modules; DOM APIs only (never innerHTML for user content) |
| Icon library | inline hand-built SVG via `buildSvg()` (settings.js) — no icon package |
| Font | Rubik (self-hosted woff2: Regular 400, SemiBold 600, Bold 700) |

Token source of truth: `assets/tokens.css` (primitive → semantic, light default + `[data-theme="dark"]`). Layout uses CSS **logical properties** (block/inline) so RTL flips for free — EXCEPT drag/overlay positioning, which MUST use **physical** `left`/`top` coordinates (repo memory `reference-rtl-logical-props-physical-coords.md`: `inset-inline-*` mirrors overlays wrongly in RTL).

## Spacing Scale

Declared values (shipped scale, used by `help.css`/`tour.css`; app chrome uses the same rem cadence). Multiples of 4:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (0.25rem) | Icon gaps, meta line gaps (`.settings-row-meta`) |
| sm | 8px (0.5rem) | Compact element spacing, reorder-row control gaps |
| md | 16px (1rem) | Default element spacing, settings-row inner gap |
| lg | 24px (1.5rem) | Settings-card padding (`.settings-row` = 24px) |
| xl | 32px (2rem) | Layout gaps |
| 2xl | 48px (3rem) | Major section breaks |
| 3xl | 64px (4rem) | Page-level spacing |

Exceptions:
- **44px minimum tap target** (`--tap-target-min`) for the drag handle, up/down arrows, rename ✎, enable toggle, and the ⓘ info button — WCAG 2.2 / iPhone touch (ORDR-01/02). Reorder rows have `min-height:44px`.
- Reorder-list rows are **compact** (~8px vertical padding inside the 44px min-height), NOT the 24px-padded `.settings-row` cards — the whole order must be scannable in one list (mockup card 1). Child (group-member) rows indent `padding-inline-start: ~28px`.
- Severity buttons are a fixed **34×34px** grid item (`.severity-button`, shipped) — an existing exception, unchanged.

## Color

Garden-green palette from `assets/tokens.css` (semantic tokens; never literal hex in new CSS). 60/30/10 read:

| Role | Value (light) | Value (dark) | Usage |
|------|---------------|--------------|-------|
| Dominant (60%) | `--color-background` #fdf8f0 (warm cream) / `--color-surface` #ffffff | #151b1b / #202828 | Page background, row surfaces |
| Secondary (30%) | `--color-surface-toggle` #f2faf5 / `--color-surface-subtle` #fef8f0 | #1c2323 | Group-header row tint, topic-row fill, ⓘ popover fill |
| Accent (10%) | `--color-primary` #2d6a4f (green-600) | #5aacac | See reserved-for list |
| Destructive | `--color-danger` #ea4b4b | (light-only by design; reserved for confirm dialog) | Not used in this phase — no destructive action (see Copywriting) |

Accent (`--color-primary`) reserved for:
- The enable-toggle pill in its ON state (`--color-primary` fill).
- The active/open ⓘ info button (border + glyph shift to primary when `aria-expanded="true"`).
- Group-header row text + the reset-order button text (`--color-primary-dark`).
- The drag-placeholder dashed outline (`--color-primary-soft`).
- Focus ring on reorder controls (`0 0 0 3px --color-green-50`, shipped pattern).

NOT accent-colored: drag handle (`--color-text-muted`), up/down/rename icon buttons (muted on `--color-surface-subtle`), disabled rows (0.5 opacity + strike-through on the label only).

**Severity scale** keeps its own shipped multi-hue ramp (green→amber→red, 0–10) via `--sev-color`; this is a domain data-encoding palette, exempt from the 60/30/10 chrome rule. The skipped-rating **—** 11th value (language-free dash pill, no translation needed) is deliberately OUTSIDE that ramp: neutral grey (`--color-sev-default-border` #ccc / dark #4a5555) so it never reads as a severity level.

## Visual Hierarchy (focal points)

- **Settings reorder list:** the eye lands on the **group-header rows** (600-weight, `--color-primary` text) that segment the list; the paired accent-text reset buttons (**Reset order** / **Reset names**) form the secondary anchor at the list's end — one quiet row, equal visual weight, no dropdown. Member rows stay visually quiet (400-weight, neutral) so structure reads before detail.
- **Restructured session form:** the **section headers** of the collapsed accordion are the anchors, with **Session topics** first (D-02 order) as the entry point; expanded content stays subordinate to its header.

## Interaction Contract (phase-specific)

Prescriptive behaviors the executor must implement; verified by ui-checker + on-device UAT (jsdom is blind to touch drag / RTL / PDF — real-device verification required, repo memory).

1. **Two-level reorder, two input methods.** Drag (pointer events — mouse + iPhone touch, `setPointerCapture`, `touch-action:none` on the handle; NOT HTML5 DnD) AND per-row up/down arrows (WCAG 2.2 baseline). Top-level items reorder among themselves; group members reorder within their group. **No cross-group moves** (D-01).
2. **Drag visuals.** Dragged row: reduced opacity (~0.55) + `--shadow-card`. Insertion point: a dashed `--color-primary-soft` placeholder. RTL: handle/arrows mirror via logical props, but drag math uses **physical** `getBoundingClientRect` top/height (not inset-inline).
3. **Arrow end-stops.** Up disabled on the first item / first group member; Down disabled on the last — dimmed to 0.3 opacity, `disabled` attribute (not just visual).
4. **Disabled rows keep their slot** in Settings (D-11); they render struck-through/dimmed but stay draggable and re-enableable in place. They simply don't render on form/export.
5. **Group header row** is the drag/reorder unit for the whole group (its own handle + arrows); it has no enable toggle and no ✎ unless renamable (group renames ship — D-05; `LOCKED_RENAME` sections keep no ✎).
6. **ⓘ severity explainer** opens on **tap/click** (mobile-safe), not hover-only; toggles `aria-expanded`; the popover is a bordered `border-inline-start:3px --color-primary` note beneath the toggle.
7. **Severity coupling (D-08).** Disabling the **Issue severity** row hides the end-of-session block AND the start-rating column inside topic rows; topics themselves remain. Its drag position controls where the end-of-session block renders in the form.
8. **Skipped rating (D-09).** 11th severity value (rendered as a language-free **—** pill, neutral grey, **first in the scale — immediately before 0**, nearest the topic so pointer travel is shortest; Ben 2026-07-23); selecting it auto-hides that topic's after-rating; the field stays mandatory (— satisfies validation). **Tab order:** coming from the topic's text field, the — pill is the FIRST Tab stop, then 0→10 — skipping a rating is one Tab + Enter/Space.
8a. **Skipped rating in export (Ben, 2026-07-22).** A topic rated — contributes NO severity to any export (no PDF bar, no markdown severity line — the topic itself still exports). If EVERY topic is —, the severity block is omitted from the export entirely, even when "Include severity before/after" is checked. Export never renders a literal "—"/N-A value.
9. **Order takes effect on next form open** (D-16); an already-open form keeps its rendered order (no live reshuffle). Within one page load, form order == export Step-1 list order == export output order (D-13 three-way invariant, asserted against SAVED order).
10. **Guided-tour anchors** (Phase 41) target the session form — verify anchors survive the D-02 restructure (decided at planning per CONTEXT discretion).
11. **Order constraint (Ben 2026-07-23, locked): Issue severity always sits after Session topics.** One declarative pairwise rule in a shared move-validator consulted by drag, arrows, AND saved-order load/import — a restored backup or migrated order is sanitized through the same rule, otherwise the invariant is theater. Enforced by affordance, never by error message: dragging Issue severity, the insertion point never appears above Session topics; dragging Session topics down past Issue severity is the same crossing and clamps identically; up/down arrows dim at the boundary exactly like the existing end-stops. Reset order (D-02 default) trivially satisfies it. All other ordering stays fully flexible.

## Accepted Exceptions (greenfield-checker calibration)

Repo memory `feedback-ui-checker-greenfield-false-positives.md`: the ui-checker's greenfield rules (2-weight max, 4px-grid-only, generic-CTA ban) mis-fire on reuse-only phases. For THIS phase, force-approve with these documented exceptions rather than spinning the loop:

- **Three font weights (400/600/700):** shipped Rubik faces already in `tokens.css`; new reorder UI uses only 400/600. Not a new decision.
- **"Reset order" CTA:** intentionally plain/literal — it is a settings utility, not a marketing CTA; the verb+noun is exact and reversible.
- **Severity multi-hue ramp:** domain data-encoding, not chrome — exempt from 60/30/10.
- All spacing values are multiples of 4; the only sub-grid fixed size (34px severity button) is pre-existing.

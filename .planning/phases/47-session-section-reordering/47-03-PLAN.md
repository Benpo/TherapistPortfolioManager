<objective>
Turn the existing Settings section list into the grouped reorder control (D-10): every row gains a drag handle + up/down arrows; groups render as header rows (their own handle/arrows + renamable title) with indented member rows; bare sections are top-level rows. Add the NEW "Issue severity" (afterSeverity) row with its ⓘ severity-off explainer (D-08), the paired Reset order / Reset names buttons (D-12), and persist the resulting order through the sectionOrder sentinel on Save (D-16). Reorder is available by pointer drag (mouse + iPhone touch) AND by accessible up/down arrows (ORDR-01/02) — both funnel through the one shared clamp.

Purpose: This is the control surface the whole feature hangs off. It must build on the shipped rename+toggle rows and their Save-staging (no new Save chrome), and it must never produce an order the shared validator would reject.

Output: settings.js grouped reorder UI + persistence; app.css styling; an arrow-reorder behavior test.
</objective>

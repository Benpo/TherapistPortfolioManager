<objective>
Build the shared data foundation the whole phase composes on: (1) a per-therapist section-order sentinel in IndexedDB modeling groups as DATA (D-05), (2) a synchronous App.getSectionOrder() cache mirroring the shipped _sectionLabelCache pattern, (3) ONE shared App.sanitizeOrder() move-validator that enforces the "Issue severity never precedes Session topics" clamp (Interaction 11) and drops unknown keys, and (4) tap-again-to-clear on the shared severity widget (D-20) so an unrated rating is reachable by deselecting the active pill.

Purpose: Every downstream consumer (Settings reorder UI, session form, both export builders, backup restore) reads order from ONE cache and validates through ONE function. Divergent sources are exactly the 260615 bug class this phase must not reintroduce. Tap-again-to-clear gives the unrated state (which the shipped validation always allowed) a reversible affordance — replacing the dropped "— (skip)" 11th-pill design (D-19/D-20).

Output: db.js sentinel path + reader; app.js default order, order cache, validator, and severity tap-again-to-clear; three behavior tests.
</objective>

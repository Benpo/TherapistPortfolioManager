<objective>
Wire the session form's severity semantics on top of the shared '— (skip)' marker (47-01) and the D-02 structure (47-04). Make skip a save-safe rating that auto-hides its end-of-session rating and shows the skipped-rating hint under the scale (D-09/G-9), and make the 'Issue severity' section's enable toggle the app-level severity switch: on new/empty sessions, off hides the end-of-session block AND the per-topic start-rating column while the topics stay (D-08); on past sessions that carry severity data, both stay visible with the disabled badge, mirroring the shipped disabled-section-with-data rule (G-11). Swap the topic-row label to 'Severity at start' so severity is named once (in Settings), never repeated on the row. validateIssues already accepts any rating state (payload.length > 0 only) — this plan must not invent new rating-mandatoriness (G-17).

Purpose: ORDR-06 (severity switch) + ORDR-07 (— skip value) on the form side. This is the second add-session pass (after the 47-04 restructure) so it stays in a later wave to avoid file conflict.

Output: skip-aware form readers + severity-off column coupling in add-session.js; the topic-row label swap; a form-severity behavior test.
</objective>

## Must-Have for v1.2

### F-01 — Full IndexedDB Encryption at Rest
**Theme:** Security | **Complexity:** L

Every client session note, severity rating, trapped emotion, and personal detail sits in plaintext in IndexedDB. A stolen device with no disk encryption exposes everything.

- PIN/passphrase entry on every app open
- Web Crypto API: PBKDF2 key derivation + AES-256-GCM encryption
- All IndexedDB writes go through an encrypt layer; all reads go through decrypt
- One-time migration: unencrypted to encrypted on PIN setup
- PIN forgotten = data gone by design. No recovery.

**Files touched:** `db.js`, `backup.js`, `add-client.js`, `add-session.js`, `overview.js`, `sessions.js`, `reporting.js`
**Dependencies:** None. Do this first — everything else builds on it.

### F-02 — Client Progress Chart (Severity Over Time)
**Theme:** Data Visualization | **Complexity:** M

Data already exists: every session carries `issues[]` with `{ name, before, after }` and a `date`. Missing: any way to see it visually.

- Per-client line chart showing average severity (before/after) across sessions
- Accessible in client detail modal
- Vanilla SVG — no library needed
- Optional: filter by issue name

**Files touched:** `overview.js`, `overview.html`
**Dependencies:** None. Works with existing data model.
**Why must-have:** A practitioner who can show severity going from 8 to 2 over six sessions is demonstrating transformation. This is the feature Sapir would demo to convince a colleague to buy.

## What to Avoid in v1.2

| Temptation | Why to Resist |
|------------|---------------|
| Cloud sync | Kills privacy value proposition, adds GDPR burden |
| AI session notes | Cloud API calls, hallucination liability |
| Scheduling/calendar | Scope creep into SimplePractice territory |
| Client portal | Adds security surface, contradicts practitioner-private model |

<objective>
Fix two ordered-list regressions in assets/pdf-export.js exposed after quick task 260522-iwr.

- **Bug A:** the typed ordinal is lost when ordered-list items are separated by paragraphs. The user types `1. X` / paragraph / `2. Y` / paragraph / `3. Z` and the PDF renders them all as "1." because parseMarkdown only groups CONTIGUOUS list lines and the renderer assigns ordinals from each block's local index.
- **Bug B:** in an RTL document, an ordered-list item whose CONTENT starts with English (LTR strong) has its "N. " prefix on the LEFT margin instead of the right, because the whole row (prefix + content) is shaped as one paragraph whose first-strong direction resolves to LTR.

Purpose: editor-1:1 export integrity. The PDF must show what the user typed. RTL list layout must remain RTL-anchored irrespective of per-item content directionality.

Output: behavior-tested fix to parseMarkdown + list renderer, plus a new falsifiable test file that fails on current code (RED) and passes after the fix (GREEN).
</objective>

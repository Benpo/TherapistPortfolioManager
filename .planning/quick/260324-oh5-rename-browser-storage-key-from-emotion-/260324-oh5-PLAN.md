2. Add a `migrateOldDB()` async function BEFORE the `openDB()` function. This handles existing users who have data under the old name. The function should:
   a. Check if an IndexedDB named `"emotion_code_portfolio"` exists using `indexedDB.databases()` (supported in all modern browsers). If `indexedDB.databases` is not available (Firefox < 126), attempt to open the old DB and check if it has object stores — if empty/version 0, skip migration.

      - Open new DB via the existing `openDB()` (which creates schema via migrations)
      - Copy all records from `clients` and `sessions` stores from old to new
      - Close both databases
      - Delete the old database: `indexedDB.deleteDatabase("emotion_code_portfolio")`
      - Log: `console.log("Migrated database from emotion_code_portfolio to sessions_garden")`
   c. If old DB does not exist, return immediately (no-op for new users)

3. Modify `openDB()` to call `migrateOldDB()` before opening the DB. Specifically, wrap the existing openDB logic: at the top of the function, before the `indexedDB.open` call, await `migrateOldDB()`. Since `openDB` already returns a Promise, convert its internals to use async/await or call migrateOldDB before constructing the Promise.

   IMPORTANT: The migration must be idempotent — if sessions_garden already has data (migration already ran), skip copying. Check by reading all clients from the new DB first; if count > 0, skip.

   IMPORTANT: Use a module-level flag `let _migrationDone = false` so migrateOldDB only runs once per page load, not on every openDB call.

4. For the Firefox fallback (no `indexedDB.databases()`): Open `"emotion_code_portfolio"` with version 1 (no upgradeneeded handler). In onsuccess, check if db.objectStoreNames.length > 0.

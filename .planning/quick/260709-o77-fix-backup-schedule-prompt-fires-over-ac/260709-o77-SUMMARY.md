# Quick Task 260709-o77: Fix Backup Schedule Prompt Fires Over Active Onboarding Tour — Summary

Added a minimal, fully typeof-guarded tour-active guard to `checkBackupSchedule()` so a DUE backup schedule check no longer stacks the Backup & Restore modal on top of an active onboarding-tour overlay (the Phase 41 release escape), while intentionally skipping the debounce-stamp write on the suppressed path so the reminder re-fires naturally after the tour closes.

## What Was Built

- **RED test** (`tests/quick-260709-o77-backup-schedule-tour-guard.test.js`): a self-contained vm-sandbox behavior test following the 25-09 idiom (Node `assert`, hand-rolled `test()` runner, `process.exit`). The `makeSandbox` factory was extended to accept a tour-state argument and mount `window.Tour = { isActive: () => active }` when supplied (left undefined otherwise). Three falsifiable cases, each with a fresh sandbox and an `openBackupModal` call-count spy:
  - **Case A** (tour ACTIVE): modal NOT opened, `portfolioBackupSchedulePromptedAt` NOT advanced, no redirect — the bug case.
  - **Case B** (tour INACTIVE, `isActive()→false`): modal opened once + stamp advances — regression guard.
  - **Case C** (`window.Tour` undefined): modal opened once, no throw — typeof-guard pass-through.
- **GREEN fix** (`assets/backup.js`): an early `return` at the top of `checkBackupSchedule()` (right after `intervalMs` resolves, before every stamp-write path) that fires when `typeof window !== 'undefined' && window.Tour && typeof window.Tour.isActive === 'function' && window.Tour.isActive()`, wrapped in try/catch. A one-line WHY comment documents the Phase 41 escape and the intentional debounce-skip.

## Test-First Proof (RED → GREEN)

- **GREEN** (after the guard): same test exits **zero** (3/3). `node tests/25-09-schedule-debounce-no-modal.test.js` still 4/4 (CR-01 write-only-on-prompt gate intact). Full suite `node tests/run-all.js` — **154 passed, 0 failed**.

## Threat Model Verification

- **T-o77-01 (DoS — reading window.Tour.isActive)**: mitigated. Full typeof-guard (window defined, `window.Tour` truthy, `isActive` is a function) + try/catch; any throw is treated as "tour inactive" so a broken/absent Tour global can never disable the backup reminder (proven by Case C).

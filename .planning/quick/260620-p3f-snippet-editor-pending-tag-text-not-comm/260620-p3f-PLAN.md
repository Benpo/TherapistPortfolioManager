# Quick Task 260620-p3f — Snippet editor: pending tag text not committed on Save

## Problem (root-caused, reproduced)

In the Text Snippets editor, adding a tag does nothing if the user types the tag
and clicks **Save** without first pressing Enter/comma/Tab. The save reports
success (the "saved" toast fires), but the tag is neither persisted nor shown.

**Root cause:** `handleSave()` reads tags via `readEditorTags()`, which only
collects already-committed `<li>` chips from `#snippetEditorTagsList`. The text
still sitting in `#snippetEditorTagsTextInput` (a typed-but-uncommitted tag) is
never flushed into a chip, so it is silently dropped. Tags are only committed by
the input's keydown handler on Enter / "," / Tab.

**Reproduced** (jsdom + fake-indexeddb harness, real `settings.js` + `db.js` +
real `settings.html`):
- Type "anxiety" → press **Enter** → Save → persisted `tags:["anxiety"]`, shown in list. ✅
- Type "depression" → click **Save** (no Enter) → persisted `tags:[]`, toast still fires. ❌ (the bug)

The placeholder hints "Add a tag and press Enter", but typing-then-Save is the
natural flow and must not lose data.

## Fix

`assets/settings.js`:
1. Add `pendingTagToCommit(committedTags, pendingRaw)` — a pure helper that
   trims + lowercases the pending input and returns the tag to append, or
   `null` when blank or a duplicate. Single source of truth for tag
   normalization.
2. Add `commitPendingTag()` — flushes the pending input into a committed chip
   (using the pure helper), then clears the input.
3. In `handleSave()`, call `commitPendingTag()` immediately before
   `readEditorTags()` so a typed-but-uncommitted tag is captured on Save.
4. Refactor the existing `addCurrent()` (keydown commit) to reuse
   `pendingTagToCommit` so Enter and Save normalize tags identically.
5. Expose `pendingTagToCommit` + `commitPendingTag` via
   `window.__SnippetEditorHelpers` for tests.

## Test (TDD RED → GREEN)

`tests/quick-260620-p3f-pending-tag-commit.test.js` (zero-dependency vm sandbox,
matches project convention — no jsdom in repo):
- **Pure:** `pendingTagToCommit` — new tag, trim+lowercase, blank/whitespace →
  null, case-insensitive dedupe.
- **Integration:** `commitPendingTag` against a capable fake DOM — a typed tag
  becomes exactly one chip with `data-tag`, input cleared; a second flush of the
  same value adds no duplicate.

Falsifiable: both helpers are `undefined` before the change (RED).

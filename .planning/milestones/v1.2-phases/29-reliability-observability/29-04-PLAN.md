<objective>
Close the two UAT-diagnosed OBS-01 gaps in the crash-log capture module. Both are already root-caused with a deterministic repro; this plan turns the diagnosis into the smallest correct fix.

GAP 1 (major, D-02 retention contract): a lost-update race in `crashlog.js append()`. Each append does an async read-all (`getEntries`) then a FULL-REPLACE write (`replaceAllCrashlog` = clear + re-add). Concurrent appends each read the pre-write set and clobber one another — last writer wins. On-device, the direct `CrashLog.logError()` seam entry (the seam Phase 28's integrity self-check uses) and the `onerror` entry were dropped every time. Repro: `tests/29-01` case 7 (5 concurrent `logError` calls, only 1/5 survives — currently RED).

GAP 2 (minor): post-load double-logging. The inline `<head>` early-buffer handler (identical across all 21 SW-registered pages) is never neutralized once `crashlog.js` loads, so an error thrown AFTER load is captured twice — once as `source:'early'`, once as its real source. `entryKey()` includes `source`, so `dedupe()` keeps both — inflating counts and burning the 50-entry budget ~2x.

Purpose: restore the OBS-01 "captures every error in a cascade" and "logged once" guarantees without regressing the 50-count ceiling, 30-day prune, mirror-survives-IDB-failure, or zero-network contract.

Output: a serialized append path + a one-line inline-handler guard replicated across 21 pages, with case 7 GREEN and a new falsifiable double-log test.
</objective>

    Fix: serialize the entire read-modify-write of `append()` behind a module-level tail-promise queue so each append's read sees the PREVIOUS append's committed write. The chain link must NEVER reject (a rejected tail would wedge all subsequent appends) — wrap the link body so failures resolve to `false`, mirroring the current never-throwing contract. Keep the existing internal steps verbatim inside the serialized link: build `combined = prune((existing||[]).concat([entry]))`, do the synchronous direct `writeMirror(combined)` FIRST (crash-survival, openDB-free), then best-effort `persistToIDB(combined)`. Preserve the absolute mirror-only backstop that already guards the merge/read path.

    Do NOT change the IDB-failure fallback semantics, the prune policy, MIRROR_DEPTH, `entryKey`, `dedupe`, or `replaceAllCrashlog`. Do NOT add any network primitive. The mirror write must remain synchronous and direct (case 5 asserts the mirror survives a thrown `indexedDB.open`, so the mirror step must not be gated behind the IDB write inside the link).

    Normalization currently happens in `append()` before the queue boundary (`var entry = normalize(rawEntry);`). Keep `normalize()` being applied to the entry that enters the link so each queued entry is a stable plain object captured by the closure (per-call `entry`, not a shared mutable).

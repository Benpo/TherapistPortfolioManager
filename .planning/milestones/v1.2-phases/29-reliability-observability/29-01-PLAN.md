<objective>
Build the OBS-01 crash-log foundation: a zero-network capture module that installs `window.onerror` + `unhandledrejection` handlers early on every app page, persists the most recent entries to IndexedDB with a localStorage mirror, prunes on write (≤30 days AND ≤50 entries), and exposes a stable `CrashLog.logError()` entry point that Phase 28's integrity self-check and the Phase 29 report screen both consume.

Purpose: Make production problems on a user's device diagnosable without any data leaving the device. This is the foundation Wave 2 (the report screen) reads from, and the seam Phase 28 (28-CONTEXT D-12) calls into.
Output: `assets/crashlog.js` (new) + an early inline head handler + a `<script src>` wiring on all 20 SW-registered app pages.
</objective>

## Artifacts this phase produces (Plan 01)

NEW symbols introduced by this plan:
- `assets/crashlog.js` — new IIFE-global module assigned to `window`/`self`/`globalThis` as `CrashLog`.
- `CrashLog.logError(entry)` — stable entry point (called by Phase 28 integrity check in Wave 2, and by the report flow). Accepts a structured entry; never throws; never makes a network call.
- `CrashLog.getEntries()` — returns the merged, most-recent-first crash entries (IDB primary, localStorage mirror as fallback when IDB is unavailable).
- `CrashLog.clear()` — empties the log (IDB store + localStorage mirror). Backs the optional Settings "clear" affordance built in Wave 2.
- `CRASHLOG_STRINGS` + `clStr(key)` — inline pre-i18n 4-language (EN/HE/DE/CS) string object + accessor, analog of `DB_STRINGS`/`dbStr`.
- A new IndexedDB object store `crashlog` (keyPath `id`, autoIncrement) added via a new migration in `db.js` — see Task 1 note on where the store is declared.
- The `crashlogBuffer` localStorage key (mirror) — guarded `localStorage.setItem`, bypasses `openDB()`.
- An early inline `<script>` head block (registered before all other page scripts) that buffers early errors into the localStorage mirror until `crashlog.js` loads.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| thrown error/stack → crash log | An uncaught error may embed client-identifying data (a client name in an error message); this enters the persisted log |
| crash log → (Wave 2) clipboard/email | The persisted log is the source the report flow will expose; capture must keep it bounded and local |
| app code → network | The capture path must never reach the network (zero-network invariant, VER-06 carry-over) |

This plan adds the must-NOT below; bounded retention (≤50 / ≤30d) limits exposure surface |

<objective>
Fix the Hebrew RTL name+date scramble (UAT test 7). At `updateSessionTitle` (add-session.js:1698) the app composes `clientName` + " • " + `dateText` into ONE un-isolated text node written to both `titleEl.textContent` and `document.title`. In Hebrew with a month-name date format, `date-format.js` intentionally returns a BARE mixed-direction string ("16 במאי 2026" — it only LRI/PDI-isolates NUMERIC formats, D-07), so an un-isolated Latin name adjacent to a bare mixed date under `html[dir=rtl]` gets reordered by the Unicode Bidi Algorithm into "2026 במאי dgh • 16". The same un-isolated " • " composition class exists at overview.js:799 (date • sessionType) and overview.js:958-961 (age/parts • Hebrew type).

Purpose: Add ONE shared First-Strong-Isolate helper and wrap each mixed run at the composition sites so mixed name/date/type lines read in logical order in Hebrew — closing a pre-existing major bidi defect. This is a display-order fix only; stored data is already correct.
Output: `window.DateFormat.isolate` helper + isolated call sites in add-session.js and overview.js + a falsifiable helper test with per-site source gates. Real Hebrew reading order and the PDF export header are human-verified.
</objective>

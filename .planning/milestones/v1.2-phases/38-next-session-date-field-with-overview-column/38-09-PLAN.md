<objective>
Close UAT test 5 (NEXT-01): a partially typed next-session date is silently discarded on save while the success toast still fires. Add a save guard on #nextSessionDate ONLY — when the native date input reports validity.badInput (a partial/unparseable entry), abort the save and surface a clear, i18n'd message instead of persisting the session without the date.

Purpose: A native `<input type="date">` exposes value="" for BOTH an empty field AND a partially entered date; the only signal that distinguishes them is validity.badInput. The save path (add-session.js:1137) reads value || "" with no partial-entry check, so a one-segment edit saves "" and reports success — the exact confusion the therapist hit in Safari.

Output: A pure, unit-tested guard wired into the single save choke point (saveSessionForm), a new toast key in 4 languages, a falsifiable RED-first guard test, and a real-Safari field-verification item (the badInput signal cannot be machine-simulated in this repo's jsdom harness).

Extends the NEXT-08 falsifiable-test discipline: because jsdom cannot set badInput on a real date input, the RED-first test exercises the extracted guard directly against stubbed validity state.
</objective>

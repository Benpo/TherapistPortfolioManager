<objective>
Fix the Hebrew RTL native date-input segment reversal (UAT test 6). In Hebrew mode every native `<input type="date">` inherits `direction: rtl` from `html[dir="rtl"]` (app.css:1289, toggled by app.js:124), and WebKit lays out the internal `::-webkit-datetime-edit` sub-fields in that inherited direction — visually reversing the browser-native mm/dd/yyyy run into yyyy/dd/mm ("2026/16/05"). Root cause and fix direction were reproduced and validated headlessly in Playwright WebKit (see debug file). No formatter change — this is a pure CSS-cascade/bidi defect.

Purpose: Restore correct, native-order date segments in Hebrew for ALL seven native date inputs in the app, closing a pre-existing major RTL defect surfaced during the Phase 38 UAT retest.
Output: A shared CSS rule set in `assets/app.css` + a source gate, plus a real-Safari human verification of segment order, value alignment, and calendar-icon side.
</objective>

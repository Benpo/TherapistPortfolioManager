|-----------|----------|-----------|----------|-------------|-----------------|
| T-37-16-01 | Tampering | `App.setLanguage` via `__afterBackupRestore` (restored `settings.language`) | low | mitigate | `setLanguage` already guards unknown languages — an unrecognized `portfolioLang` falls back to `I18N_DEFAULT` (app.js:117-118); no arbitrary code path from the restored value. |
| T-37-16-02 | Tampering | `App.applyTheme` via `__afterBackupRestore` (restored `settings.theme`) | low | mitigate | `applyTheme` sets `data-theme='dark'` only when the value is exactly `'dark'`, otherwise removes the attribute — an arbitrary theme string cannot inject markup or an unexpected attribute value. |
| T-37-16-03 | Tampering | backup.js restore of `dateFormat`/`sessionTypes` (never-clobber) | low | accept | Restore only writes non-empty values into two scalar localStorage keys already consumed by validated engines/resolvers; never-clobber removes the removeItem path, reducing (not adding) write surface. |

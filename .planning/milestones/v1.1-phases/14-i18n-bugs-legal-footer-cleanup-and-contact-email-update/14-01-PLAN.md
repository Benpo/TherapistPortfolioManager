**D-01: Language persistence bug fix**

Root cause: When the disclaimer page redirects to the app after acceptance, it uses `window.location.replace` without preserving the language. The app's `index.html` redirects to `disclaimer.html` or `license.html` but those pages may not read `portfolioLang` from localStorage correctly on redirect.

In `assets/disclaimer.js`, find the acceptance redirect logic (the section that runs after the user clicks accept). Ensure that before redirect, the current language is saved to localStorage:
```js
localStorage.setItem('portfolioLang', currentLang);
```
This should already exist but verify it fires BEFORE the `window.location.replace` call. If the disclaimer page redirects to `index.html` and index.html then redirects to disclaimer/license, the chain must preserve `portfolioLang` in localStorage throughout.

Also in `assets/disclaimer.js`, in the `detectLang()` function, ensure the URL param `?lang=` is checked first (it already does this). The key fix: when the landing page links to `disclaimer.html?readonly=true`, it should append `&lang=` with the current language. In `assets/landing.js`, find the footer-terms link setup (around line 579 `setText2('footer-terms', ...)`). After `applyLang` sets the footer terms text, also update the href:
```js
var termsLink = document.getElementById('footer-terms');
if (termsLink) termsLink.href = './disclaimer.html?readonly=true&lang=' + lang;
```

  <name>Task 2: Canvas crop logic with drag, zoom, confirm/cancel</name>
  <files>assets/add-client.js</files>
  <action>
Modify `assets/add-client.js` to implement the crop modal logic. All new code goes inside the existing DOMContentLoaded handler.

1. **Crop state variables** (at top of DOMContentLoaded, near `let photoData`):
   ```
   let cropImage = null;  // Image object loaded from file
   let cropScale = 1;
   let cropOffsetX = 0, cropOffsetY = 0;
   let isDragging = false;
   let dragStartX = 0, dragStartY = 0;
   ```

2. **DOM references** for crop modal elements:
   cropModal, cropCanvas (get 2d context), cropClose, cropZoomSlider, cropZoomIn, cropZoomOut, cropConfirmBtn, cropCancelBtn, recropBtn.

3. **openCropModal(dataURL)** function:
   - Create new Image(), set src to dataURL, onload:
     - Store as cropImage
     - Calculate initial scale so image covers the 300x300 canvas area (fit-cover): `Math.max(canvasW / img.naturalWidth, canvasH / img.naturalHeight)`
     - Set cropScale to this base scale, cropOffsetX/Y to center the image
     - Set slider min to this base scale, max to base * 3, value to base
     - Call drawCrop()
     - Show modal: cropModal.classList.remove("is-hidden")
     - Apply translations: App.applyTranslations(cropModal)

4. **drawCrop()** function:
   - Set canvas width/height to 300 (or use devicePixelRatio for sharp output: canvas.width = 300 * dpr, canvas.height = 300 * dpr, then ctx.scale(dpr, dpr))
   - Clear canvas
   - Draw cropImage at (cropOffsetX, cropOffsetY) with dimensions (naturalWidth * cropScale, naturalHeight * cropScale)

5. **Drag handlers** on crop canvas container (use pointer events for unified mouse+touch):
   - pointerdown: set isDragging=true, record dragStartX/Y (evt.clientX - cropOffsetX), canvas container cursor = "grabbing"
   - pointermove: if isDragging, cropOffsetX = evt.clientX - dragStartX, cropOffsetY = evt.clientY - dragStartY, drawCrop()
   - pointerup/pointercancel: isDragging=false, cursor = "grab"

6. **Zoom handlers**:
   - Slider input: cropScale = parseFloat(slider.value), recenter image proportionally, drawCrop()
   - ZoomIn button: increase slider by 0.2 (clamped to max), trigger input event
   - ZoomOut button: decrease slider by 0.2 (clamped to min), trigger input event
   - Wheel event on canvas container: adjust zoom by deltaY, prevent default, update slider

7. **Confirm button**:
   - Create offscreen canvas at output resolution (300x300 pixels — good quality for avatars)
   - Draw the same crop state onto it
   - Extract: `offCanvas.toDataURL("image/jpeg", 0.85)` — JPEG at 85% quality to keep size small
   - Set `photoData` to this result
   - Update photoPreview.src, show preview, show recropBtn
   - Close modal

8. **Cancel button / close button / overlay click**:
   - If this was a NEW upload (no previous photoData): revert — clear photoData, hide preview, hide recropBtn, reset file input
   - If this was a RECROP (photoData existed before): just close modal, keep old photoData
   - Use a `let cropIsRecrop = false` flag set when opening

9. **Hook into photoInput change handler** (line 48-57):
   - REPLACE the direct photoData assignment. After `readFileAsDataURL(file)`, call `openCropModal(rawDataURL)` instead of setting photoData directly. The confirm handler sets photoData.

10. **Recrop button**:
    - On click: call `openCropModal(photoData)` with `cropIsRecrop = true`
    - Show recropBtn whenever photoPreview is visible (add show logic after confirm and after loading existing client photo)

11. **Existing client photo edit mode**: When editing an existing client (line 67-117), if photoData exists, show recropBtn.

IMPORTANT: No external libraries. Pure Canvas API. Keep all crop code in add-client.js — no separate file needed for this amount of code.

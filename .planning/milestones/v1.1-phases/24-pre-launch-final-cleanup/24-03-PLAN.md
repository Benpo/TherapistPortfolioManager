<objective>
Item 2 (CONTEXT D-02 through D-07): Add a Cancel / Revert affordance to the edit-session screen that lets the user undo unsaved edits and return to read mode in place — composing cleanly with the read-mode-is-default story established in Phase 22. Rename the overview clock-icon expansion's "Edit" button to "View" (with pencil icon) to signal read mode is the default entry.

Both halves share the Hebrew i18n string-locking work (D-05), so they are planned together. Both compose with Plan 01's `populateSpotlight` (no functional dependency — just touches same files).

**Decisions implemented:**
- **D-02 (position):** Cancel button lives in the existing `.inline-actions` row alongside Save / Delete / Back to Overview. No new sticky-footer pattern.
- **D-03 (confirm rule):** Confirm dialog only when `window.PortfolioFormDirty?.()` returns true. Clean form → silent revert.
- **D-04 (asymmetric labels):** Visible label swaps: `Cancel` (clean) / `Discard changes` (dirty).
- **D-05 (Hebrew noun/infinitive):** All Hebrew strings use noun/infinitive form per the project-wide rule.
- **D-06 (lifecycle):** Cancel reverts to last-saved snapshot AND switches form to read mode. No navigation. No reload.
- **D-07 (companion):** Overview clock-icon expansion's "Edit" button renamed to "View" + pencil icon.

Purpose: Close the major UAT gap. Without Cancel/Revert, therapists who accidentally edit a session have no clean way back — only Save (commits) or Delete (destructive) or Back to Overview (loses unsaved changes silently). Phase 22 round-3 UAT marked this "major" — required for end-user E2E UAT readiness.

Output: New Cancel button in add-session.html, dirty-state-aware revert handler in add-session.js, renamed "View" label in overview, fully locale-locked i18n strings.
</objective>

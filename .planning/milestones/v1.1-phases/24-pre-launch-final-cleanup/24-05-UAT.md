# Plan 24-05 — Browser UAT (Snippet Settings UI)

**Why this exists:** the Snippets Settings surface is mostly modal UI — list rendering with em-dash preview, chip input with autocomplete, collision modal, RTL flip, 4-locale rendering, cross-tab BroadcastChannel sync. These behaviors are gated by 31 unit-test scenarios on the 5 pure-function helpers, but the visual + interactive surface needs human verification.

**Pre-conditions:**
- Local dev server running (`python3 -m http.server 8080` or equivalent).
- Browser console open.
- Hard-reload (`Cmd+Shift+R` / `Ctrl+Shift+R`) to pick up the new sw.js cache version.
- At least one client in the DB so the rest of the app is navigable.
- Open `settings.html` for the UAT.

**Task 1 status rule:** every scenario below must produce the expected outcome. If even one fails, Plan 05 Task 1 stays `in_progress` — do NOT mark Plan 05 complete. Re-run scenarios after fixing each defect.

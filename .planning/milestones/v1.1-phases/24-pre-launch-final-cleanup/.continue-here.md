## Anti-pattern protocol (still binding for future phases)

Two rules from the 24-01 incident remained binding through Phase 24 and prevented zero recurrences:

1. **No grep-gate-only verification for runtime-behavior code.** Behavior test FIRST (RED), production code SECOND (GREEN), `node` output shown verbatim before commit.

2. **Identifier-resolution-chain trace for scope-crossing edits.** Local → closure → module → `window.<named-element>` → undefined. Explicit `window.` prefix in source.

A third pattern surfaced in session 3 — **"is this function idempotent?"** — caught the duplicate-theme-button bug (`initCommon` called twice from two DCL handlers). Add to the checklist for Phase 25+.

## Conventions worth carrying forward

- **Pre-commit hook auto-bumps `sw.js CACHE_NAME`** on every asset commit — EXCEPT when sw.js itself is in the diff (then manual `chore: bump CACHE_NAME` follow-up required). Hook fired 13 times this session: v124 → v137.
- **`origin/main` is the live-app branch.** Local main currently 50 commits ahead. Push gated on Ben's explicit OK per session.
- **Hebrew noun/infinitive forms** for action labels (D-05). Toast verbs use passive past tense (existing convention, e.g., "ההשלמה נשמרה").
- **Unicode-aware regex** (`\p{L}\p{N}` with `u` flag) for any user-typed input that may contain non-Latin characters. Bounded quantifiers keep ReDoS safety intact.
- **L3 IP-strip discipline.** Seed content owned by Sapir is fine to ship; chart-cell tags / cell-based ids reference Dr. Bradley Nelson's published structure and were stripped. Rule: when a seed pack mirrors an external IP structure, store the data without the structural metadata; user-added tags fill the gap.

## Visual Hierarchy

### Passphrase Prompt Modal

Primary anchor: the heading "Create a backup passphrase" (Heading role, 20px/600, centered or left-aligned at top of modal).

Hierarchy order (top to bottom):
1. Heading — primary anchor, establishes task context.
2. Body copy — one sentence explaining the irreversibility consequence (warning background `--color-warning-bg`, `--color-warning-text`).
3. Passphrase input field — receives focus on modal open.
4. Confirm passphrase input field.
5. Inline error (if mismatch) — `--color-danger`, appears below confirm field.
6. Action row — Confirm button (accent, right-aligned or full-width), Skip encryption button (text style, left-aligned or secondary position).

The confirm button is the visual terminus; the user's eye travels top-to-bottom from task framing to action.

### Security Guidance First-Launch Note

Primary anchor: the heading "Your data lives here — keep it safe." (Heading role, 20px/600).

Hierarchy order:
1. Heading — anchors the message.
2. Body copy — explains consequence in plain language.
3. "I understand" dismiss button — single CTA, full-width on mobile, right-aligned on desktop.

The note renders at the top of the first app screen after activation, above all other content, so the heading is the first element the user encounters.

## Interaction Contracts

### Courtesy Translation Banner

- Renders at top of `<body>` inside `.legal-content`, before any legal prose.
- `<aside role="note">` — not a landmark, not dismissable.
- Link to German version opens same tab (sibling file, e.g., `impressum.html`).
- Not shown on DE files.

### Language Switcher on Legal Pages

- Navigates to sibling language file (e.g., `impressum-en.html` → `impressum-he.html`).
- Active language highlighted in switcher.
- Existing `globe-lang.js` reused; each legal file configures its own sibling links.

### "Already bought?" Auto-detection (landing.html)

- On click: synchronously check `localStorage.getItem('portfolioLicenseActivated')` AND `localStorage.getItem('portfolioLicenseInstance')`.
- If both present: show `.backup-reminder-banner`-style banner with "Active license detected. Taking you to the app." then `setTimeout(2000, () => window.location = 'index.html')`.
- If not both present: navigate to `license.html` immediately (existing behavior).

### License Page Context-Aware Chrome

- On page load: call `isLicensed()` from `assets/license.js`.
- If `true`: render `.app-header` + `.app-nav` (matching app pages). "Home" link = `index.html`.
- If `false`: render `.legal-topbar` (matching legal pages). "Home" link = `landing.html`.
- Both states use existing CSS — no new layout components.

### Gate Hardening (5 app pages)

- Inline gate script checks BOTH `portfolioLicenseActivated` AND `portfolioLicenseInstance`.
- Silent redirect to `license.html` on failure — no user-visible error state.
- No UI change — interaction contract is the redirect itself.

### Passphrase Prompt Modal

- Triggers on "Export Encrypted Backup" click.
- Two fields: passphrase + confirm passphrase.
- Validation: fields must match, minimum 1 character.
- Mismatch: inline error below confirm field (red text, `--color-danger`), fields cleared.
- Confirm button ("Encrypt and save"): disabled until both fields match and are non-empty.
- Cancel button ("Skip encryption"): closes modal, no backup created.
- Uses existing modal pattern from `assets/app.css`.

### Security Guidance Notes

- First-launch note: shown once on first app load after activation. Dismissable with "I understand" button. NOT shown again after dismiss.
- Backup reminder banner: shown on schedule per existing backup reminder logic. Body copy updated to include data-loss consequence (see Copywriting above).
- Settings/Help section: always visible, not dismissable. Persistent location as required by D-23.

App.lockBodyScroll = function() {

**Part B — Overlay-close behavior (per D-03):**

Two categories of modals:

1. **Read-only/info modals** (client detail modal on overview page, confirm dialogs): Overlay tap CLOSES the modal.

2. **Form modals with unsaved data** (edit-client modal, crop modal, passphrase modal): Overlay tap is DISABLED (do nothing on overlay click). These modals can only be closed via their Cancel/X button.

Implementation: Add click handlers on `.modal-overlay` elements.

For read-only modals (client detail in overview.js):
```javascript
overlay.addEventListener("click", () => {
  modal.classList.add("is-hidden");
  App.unlockBodyScroll();
});
```

For form modals: either don't add a click handler on overlay, or add one that does nothing / shows a brief visual hint that overlay-close is disabled.

For confirm dialogs (`.confirm-card` modals): overlay click = cancel action (same as clicking Cancel button).

DO NOT add "discard changes?" confirmation dialogs — per D-03, the simpler approach of disabling overlay-close on form modals is acceptable (the "or" in the decision). The confirmation dialog adds complexity that can be deferred.

**Gathered:** 2026-03-15

<domain>
## Phase Boundary

Redesign the landing page visual identity — colors, typography, layout, botanical illustrations, and overall look and feel. The page structure, content, pricing, and i18n are already done (Phase 5). This phase changes only HOW it looks, not WHAT it says or in what order.

<decisions>
## Implementation Decisions

### Color Palette & Mood
- Keep the garden green + warm cream base palette (same as app)
- Elevate it to feel **premium and natural** — not budget, not cold corporate
- Hero section: gradient from darker green to lighter cream — creates visual separation from rest of page
- Hero must feel distinct from the sections below — the botanical illustrations help achieve this
- CTA button stays green with subtle shadow — refined, not aggressive
- Overall mood: somewhere between Calm/Headspace warmth and Apple cleanliness — natural, premium, inviting

### Botanical Illustrations — Style (CRITICAL)
- **Hand-drawn doodle style, black line art** — NOT geometric/minimalist SVG like current
- Reference images provided by Sapir: garden doodles with flowers, watering cans, potted plants, garden tools, leaves, butterflies
- Characteristics: thin black outlines, organic hand-drawn feel, detailed but not cluttered, warm and personal (like sketches in a notebook)
- NO color fill — black lines only (on light mode). Dark mode: invert to white/cream lines
- Must feel like a real person drew them — personal, crafted, not "designed by algorithm"

### Botanical Illustrations — Placement
- **Claude's Discretion with mockups** — create multiple placement variations for Sapir to choose from:
  - Option A: Hero + doodle section dividers (Claude's recommendation)
  - Option B: Scattered throughout the page
  - Option C: Subtle background pattern across entire page
- Sapir will pick after seeing visual results

### Feature Card Icons
- Replace emoji icons (📁 🔒 📴 ✨ 💰 📝) with **small doodle illustrations** matching the botanical style
- Each card gets a relevant hand-drawn icon (e.g., folder with leaves, lock with vine, etc.)
- Must maintain the garden/botanical theme consistency

### Section Dividers
- Replace current geometric SVG divider with **doodle-style dividers**
- Small botanical elements: a branch, a flower, a leaf — hand-drawn style
- Consistent with the overall illustration language

### Claude's Discretion
- Typography: font sizes, weights, spacing, hierarchy — Claude decides (Rubik stays as the font)
- Card/section layout: grid columns, card styling, whitespace, padding — Claude decides
- Exact doodle illustration designs — Claude creates SVG doodles in the hand-drawn style
- Hero gradient exact colors and direction
- Dark mode treatment of illustrations (white/cream lines on dark)
- Mobile responsive behavior of illustrations
- Whether illustrations need to be simplified on mobile

</decisions>

<specifics>
## Specific Ideas

### Illustration Style References
Sapir provided 4 reference images showing the desired illustration style:
1. **Detailed garden doodles** — flowers in vases, watering cans, boots with flowers, garden tools, birds, butterflies. Thin black lines, vintage hand-drawn feel
2. **Bold garden doodle set** — person watering plants, gardener with tools, wheelbarrow, spray bottle. Slightly thicker lines, playful cartoon style with some black fills
3. **Plant doodles notebook** — potted plants, succulents, cacti in various pots. Classic bullet-journal doodle style, thin Micron pen aesthetic
4. **Vintage garden tools** — boots, rake, hat, gloves, watering can, flowers. Cross-hatching and texture details, slightly more formal illustration style

**Common thread across all references:** black line art, botanical/garden theme, organic hand-drawn quality, no color fills, warm and inviting personality

### Key Design Principles from Sapir
- Must feel **natural AND premium** simultaneously
- Botanical illustrations are important to the identity — not optional decoration
- The page should feel like a product made with care and personality, not a template
- App icon reference: white leaf on green rounded square (image.png in repo root)

</specifics>

## Existing Code Insights

### Reusable Assets
- `assets/tokens.css`: Full design token system with garden palette + dark mode — landing page uses these
- `assets/landing.css` (706 lines): Complete landing page styles — this is the primary file to modify
- `assets/landing.js` (590 lines): i18n system with 4 languages — feature icon text lives here
- `landing.html` (466 lines): Full page structure with SVG illustrations inline

### Established Patterns
- CSS custom properties (`var(--color-primary)`, etc.) used throughout — new styles must use tokens
- `color-mix()` function used for opacity effects — established pattern
- CSS logical properties (inline-start, block-end) for RTL support — must maintain
- `[data-theme="dark"]` selector for dark mode overrides
- Hero botanical SVGs are inline in HTML — new illustrations will likely follow same pattern
- Botanical divider is a standalone `<div class="botanical-divider">` between sections

### Integration Points
- SVG illustrations are inline in `landing.html` — replacing means editing HTML directly
- Feature card emojis are in both HTML (initial render) and `landing.js` LANDING_I18N (language switch)
- Section divider is a single reusable botanical SVG component
- Dark mode must work for all new illustrations — current SVGs use `currentColor`

### Key Constraints
- Vanilla HTML/CSS/JS only — no build tools, no npm
- SVG illustrations must be inline (no external image files) for offline PWA support
- All changes must maintain RTL (Hebrew) support via logical properties
- 4-language i18n must continue working — icon changes need updates in landing.js

</code_context>
